# v1.5 Roadmap — i18n MDX leve

## Contexto

v1 ships com:
- UI traduzida em 11 idiomas (Header, Footer, pages, components)
- Lições MDX traduzidas no repo (`content/i18n/<locale>/`) mas **servidas só em PT-BR**
- Footer com versão + timestamp + commit SHA
- Build estático leve (~150 páginas, ~250KB First Load JS)

## Problema técnico

`LocalizedMdx` (client-side swap) e per-locale routes (`/[locale]/...`)
**ambos sofrem do mesmo OOM em build**: webpack cria 1 chunk JS por
`import('content/...mdx')`. Com 124 paths × 11 locales = 1.240 chunks,
o webpack pré-processa todos os MDX (parse + LaTeX→KaTeX + JSX→React)
em paralelo no heap. → OOM mesmo com 8GB.

## Solução real: bypass webpack pra MDX traduzido

Não bundlar MDX traduzido como JS chunk. Em vez disso:

1. **Build-time**: pra cada `(locale, path)` em `generateStaticParams`,
   ler o arquivo MDX do filesystem, compilar com `@mdx-js/mdx`
   programaticamente, e renderizar pra HTML string usando
   `react-dom/server`.

2. **Page component**: emite o HTML diretamente via
   `<div dangerouslySetInnerHTML={{__html: html}} />`. Zero chunks
   webpack pra translation MDX. Apenas o HTML estático fica em `out/`.

3. **Per-locale routes**: `app/[locale]/[categoria]/[...caminho]/page.tsx`
   gera HTML separado por locale. Cada usuário baixa só o seu locale.

### Stack adicional
- `@mdx-js/mdx` (compilador programático)
- `remark-math` + `rehype-katex` (já temos para PT-BR, reaproveitar)
- `react-dom/server`'s `renderToStaticMarkup`

### Estimativa
1-2 horas de trabalho concentrado:
- 30min: setup pipeline programático MDX → HTML
- 30min: criar `app/[locale]/[categoria]/[...caminho]/page.tsx`
  com `generateStaticParams` lendo `content/i18n/`
- 20min: atualizar `LocaleSwitcher` pra navegar pra `/[locale]/...`
- 20min: testar build, validar OOM resolvido, ajustar memória
- 10min: deploy + smoke test

## Alternativas consideradas

| Opção | Build OOM | Cliente leve | URLs |
|---|---|---|---|
| LocalizedMdx (atual) | ❌ OOM | ❌ baixa todos chunks | mesmas |
| Code-split por locale (1 chunk/lang) | ⚠️ talvez | ⚠️ ~5MB/idioma | mesmas |
| **Per-locale + filesystem read (escolhida)** | ✅ ok | ✅ HTML estático puro | `/[locale]/...` |
| JSON fetch + client render | ✅ ok | ⚠️ flash + JS render | mesmas |
