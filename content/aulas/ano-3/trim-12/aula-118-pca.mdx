### Setup

Dados $X \in \mathbb{R}^{N \times d}$ ($N$ amostras, $d$ features). Centralize: $\tilde X = X - \mathbf{1} \bar x^T$ com $\bar x = \frac{1}{N}\sum x_i$.

### Matriz de covariância amostral

$$\Sigma = \frac{1}{N - 1} \tilde X^T \tilde X \in \mathbb{R}^{d \times d}$$

Sempre simétrica e PSD.

### Decomposição espectral

$\Sigma = V \Lambda V^T$ com $V$ ortogonal, $\Lambda = \text{diag}(\lambda_1, \ldots, \lambda_d)$, $\lambda_1 \geq \cdots \geq \lambda_d \geq 0$.

### Componentes principais

- **Vetor principal** $k$: $v_k$ — coluna $k$ de $V$.
- **Score** da amostra $x_i$ na PC $k$: $z_{ik} = v_k^T (x_i - \bar x)$.
- **Variância explicada** pela PC $k$: $\lambda_k$.
- **Proporção**: $\lambda_k / \sum_j \lambda_j$.

### Otimalidade

A PC 1 maximiza variância da projeção:

$$v_1 = \arg\max_{\|v\| = 1} v^T \Sigma v$$

A PC 2 maximiza dentre as ortogonais a $v_1$, e assim por diante.

### Conexão com SVD

Se $\tilde X = U \Sigma_{SVD} V^T$ (SVD da matriz centralizada):

- Componentes principais = colunas de $V$.
- Scores = $U \Sigma_{SVD}$.
- Autovalores de $\Sigma$: $\lambda_k = \sigma_k^2 / (N - 1)$.

**SVD da matriz $\tilde X$ é mais estável numericamente** que decompor $\Sigma$ explicitamente.

### Reconstrução

Mantendo $K$ componentes:

$$\hat x_i = \bar x + \sum_{k=1}^K z_{ik} v_k$$

Erro Frobenius mínimo entre todas projeções de posto $K$ (Eckart-Young).

Imagina que você tem uma **caixa de bolinhas espalhadas** em forma de **charuto**. O charuto é comprido pra um lado e fininho pro outro. **Componente principal** é só essa "**direção do charuto**" — o jeito mais importante de ver as bolinhas.

Se eu te peço pra **explicar a posição das bolinhas com um número só**, o melhor número é "**onde está ao longo do charuto**". As outras direções não importam tanto.

<Porta nivel="10" titulo="Pra criança de 10 anos">

Você tem **dados de altura e peso** de 1.000 pessoas. Cada pessoa é um ponto em 2D (altura, peso). Mas as duas variáveis são **bem correlacionadas**: pessoa alta geralmente é mais pesada.

Os dados se distribuem em uma **nuvem alongada** — tipo um charuto inclinado. PCA encontra:

- **Direção principal**: a direção do charuto (mistura de altura+peso).
- **Direção secundária**: perpendicular, captura "**desvio do padrão**".

Se você projeta cada pessoa só na **direção principal**, perde pouca informação — a maior parte da variabilidade está ali. Você reduziu de 2D pra 1D.

Em problemas reais com 100 ou 10.000 features, PCA encontra as **5-10 direções principais** que capturam quase toda a variação. **Reduz drasticamente** o tamanho dos dados.

</Porta>

<Porta nivel="15" titulo="Pra adolescente de 15 anos">

PCA é a aplicação mais utilizada da álgebra linear em ML, estatística e ciência de dados. **Toda análise exploratória de dados de alta dimensão começa com PCA**.

### Receita prática

1. **Centralize**: $\tilde X = X - \bar X$.
2. **Calcule SVD**: $\tilde X = U \Sigma V^T$.
3. **Componentes**: colunas de $V$.
4. **Scree plot**: $\sigma_k^2$ em função de $k$.
5. **Escolha $K$**: corte onde scree "joelha" ou onde variância cumulativa $\geq 95\%$.
6. **Projete**: $Z = \tilde X V_K = U \Sigma_K$.

### Variância explicada

$$\text{ExpVar}(k) = \frac{\lambda_k}{\sum_j \lambda_j} = \frac{\sigma_k^2}{\sum \sigma_j^2}$$

Cumulativa: $\sum_{k=1}^K \text{ExpVar}(k)$.

Regra de polegar: 80% para visualização, 95% para análise quantitativa.

### Quando funciona bem

- Variáveis correlacionadas (covariância forte).
- Estrutura "linear" nos dados.
- Direções de variância importam (não a estrutura local).

### Quando NÃO funciona bem

- Estrutura **não-linear** (use t-SNE, UMAP, autoencoders).
- Dados com **outliers** fortes (use Robust PCA).
- Variáveis em escalas muito diferentes (sempre **padronize antes**!).
- Quer **classificação** (use LDA — supervisionado).

### Caso famoso: Iris dataset (Fisher 1936)

4 features (comprimento/largura de pétala/sépala). PCA reduz a 2D mantendo 95%+ da variância. Visualização revela 3 clusters de espécies — Fisher já mostrava isso 90 anos atrás.

</Porta>

<Porta nivel="25" titulo="Pra estudante de engenharia">

PCA é a **primeira ferramenta** de qualquer cientista de dados. Familiarize-se profundamente.

### Algoritmo numérico padrão

```python
import numpy as np
from sklearn.decomposition import PCA

# Padronizar (importante quando features em escalas diferentes!)
X_std = (X - X.mean(axis=0)) / X.std(axis=0)

# PCA via sklearn (usa SVD internamente)
pca = PCA(n_components=10)
Z = pca.fit_transform(X_std)
print("Variância explicada:", pca.explained_variance_ratio_)
print("Cumulativa:", np.cumsum(pca.explained_variance_ratio_))
```

### Aplicações canônicas

**Genômica** (microarray): 30.000 genes × 100 pacientes. PCA reduz a ~10 dimensões → cluster de subtipos de câncer (Golub et al. 1999).

**Finanças**: yield curves de bônus em diferentes maturidades — 3 PCs (level/slope/curvature) capturam ~99% da variância. **Litterman & Scheinkman 1991**: base de fixed income trading.

**Imagem (Eigenfaces)**: rostos centralizados, PCA → bases ortonormais. Reconhecimento via projeção (Turk-Pentland 1991).

**Speech**: MFCC features + PCA pra reduzir.

**Bioinformática**: 1000 Genomes Project — PCA dos alelos revela estrutura geográfica (cluster por continente).

**Sensores**: redução de canais em arrays de sensores correlacionados.

### Pipeline ML típico

```
raw data → standardize → PCA → modelo (kNN, SVM, NN, ...)
```

Reduz dimensão → reduz overfitting + acelera treino.

### Variantes importantes

- **Kernel PCA**: PCA no espaço de features induzido por kernel — capta não-linearidade.
- **Sparse PCA**: $\ell_1$ nos loadings → componentes interpretáveis.
- **Robust PCA**: tolera outliers (Candès-Li-Ma-Wright).
- **Incremental PCA**: streaming.
- **Probabilistic PCA** (Tipping-Bishop): visão Bayesiana, lida com missing data.

### Padronização: pegadinha clássica

Se você tem features em escalas diferentes (ex: idade em anos, salário em R\$), **sem padronizar** o salário domina toda a variância. PCA encontra "direção do salário". **Sempre padronize** primeiro (z-score) antes de PCA, salvo razão específica em contrário.

### Ortogonalidade dos componentes

PCs são ortogonais por construção. Score $z_k$ tem $\text{Cov}(z_j, z_k) = 0$ para $j \neq k$. Isto facilita interpretação e modelagem subsequente (sem multicolinearidade).

### Hotelling 1933

Harold Hotelling formalizou PCA em [*Analysis of a Complex of Statistical Variables into Principal Components*](https://psycnet.apa.org/record/1934-00645-001) — Journal of Educational Psychology. Análise inicial sobre variáveis psicométricas. 90+ anos depois, ainda é técnica padrão.

<Porta nivel="40" titulo="Pra profissional sênior">

### Geometria estatística

PCA é projeção em **subespaço afim de menor dimensão** que minimiza erro quadrático médio. Equivalente a:

- Maximizar variância (visão de Hotelling).
- Minimizar reconstrução (visão de Pearson 1901).
- Maximizar likelihood Gaussiana (Probabilistic PCA — Tipping-Bishop 1999).

Os três visões dão a mesma solução.

### Identifiability

PCs são únicos a menos de:
- **Sinal**: $v_k$ ou $-v_k$.
- **Permutação dentro de espaço próprio degenerado** (autovalores iguais).

Quando $\lambda_k \approx \lambda_{k+1}$, PCs são instáveis — pequena perturbação muda muito.

### Random matrix theory

Para covariância amostral de dados Gaussianos i.i.d., autovalores têm distribuição limite **Marchenko-Pastur** ($N, d \to \infty$, $d/N \to c$):

$$\rho(\lambda) = \frac{\sqrt{(\lambda_+ - \lambda)(\lambda - \lambda_-)}}{2\pi c \lambda}$$

com $\lambda_\pm = (1 \pm \sqrt c)^2 \sigma^2$. Útil para distinguir **sinal vs ruído** em dados de alta dimensão. **BBP transition** (Baik-Ben Arous-Péché): autovalores acima do bulk ↔ sinal real.

### Conexão com PageRank, Laplacian, Diffusion Maps

Spectral clustering, diffusion maps e MDS clássico são parentes próximos. Todos diagonalizam matrizes simétricas/PSD construídas dos dados.

### Conexão com BS (Aula 119)

Em **portfólios multi-asset**, PCA da matriz de covariância de retornos:
- **PC1**: fator "mercado" (correlação geral).
- **PC2-PC3**: fatores setoriais.
- **PCs altos**: idiossincráticos, tipicamente filtráveis como ruído.

Esta é a base de **factor models** (Fama-French, Barra) usados em risk management institucional.

### Estado da arte 2020s

- **Sparse + structured PCA**: combinação de regularização para componentes interpretáveis.
- **Tensor PCA**: para dados multimodais (imagens × tempo × espacial).
- **Distributed PCA**: federated learning, MapReduce.
- **PCA via diffusion**: difusão estocástica para dados em variedades.
- **Differentially private PCA**: garantias de privacidade.

### Limitações fundamentais

PCA captura **variância**, não **discriminação**: dois clusters distintos podem ser mal-separados após PCA se a direção que os separa tem pouca variância. Para classificação supervisionada, use **LDA** ou **t-SNE/UMAP** + classificador.

</Porta>

<Porta nivel="pratica" titulo="Aplicação prática">

### Receita pronta-pra-usar

```python
import numpy as np
from sklearn.preprocessing import StandardScaler
from sklearn.decomposition import PCA
import matplotlib.pyplot as plt

# 1. Carregue dados
# X = ... (N amostras, d features)

# 2. Padronize (CRÍTICO se features em escalas diferentes)
scaler = StandardScaler()
X_std = scaler.fit_transform(X)

# 3. PCA
pca = PCA(n_components=min(X_std.shape))
Z = pca.fit_transform(X_std)

# 4. Scree plot
plt.plot(np.cumsum(pca.explained_variance_ratio_))
plt.axhline(0.95, color='r', linestyle='--')
plt.xlabel('Número de componentes')
plt.ylabel('Variância cumulativa explicada')

# 5. Escolha K (95% por exemplo)
K = np.searchsorted(np.cumsum(pca.explained_variance_ratio_), 0.95) + 1

# 6. Reduza
pca_K = PCA(n_components=K)
Z_K = pca_K.fit_transform(X_std)
```

### Checklist prático

- [ ] **Padronizou**? Senão, features de escalas grandes dominam.
- [ ] **Outliers** removidos ou usar Robust PCA?
- [ ] **Missing values** tratados? (impute, IPCA, ou descarte).
- [ ] **Visualizou scree plot** antes de escolher $K$?
- [ ] **Avaliou variância cumulativa**?
- [ ] **Interpretou loadings** (colunas de $V$)?

### Variantes prontas

```python
from sklearn.decomposition import (
    PCA, KernelPCA, IncrementalPCA, SparsePCA, TruncatedSVD
)
```

- `PCA`: padrão.
- `KernelPCA`: não-linear via kernel.
- `IncrementalPCA`: dados grandes/streaming.
- `SparsePCA`: loadings esparsos.
- `TruncatedSVD`: para matrizes esparsas (LSA).

### Quando preocupar?

- **Componentes não interpretáveis**: combinações de muitas features. Use Sparse PCA ou ICA.
- **Decay lento da variância**: dados sem estrutura linear forte. PCA não vai ajudar muito.
- **Sensibilidade a outliers**: 1 amostra anormal pode mover a PC inteira. Use RobustPCA.
