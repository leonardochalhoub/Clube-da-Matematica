<Porta nivel="formal" titulo="Decomposição espectral">

### Definição

$A \in \mathcal{M}_n(K)$ é **diagonalizável** se existe $P$ inversível tal que:

$$A = PDP^{-1}, \quad D = \text{diag}(\lambda_1, \ldots, \lambda_n)$$

### Condições equivalentes

São equivalentes:

1. $A$ é diagonalizável.
2. $A$ tem $n$ autovetores LI.
3. Soma das dim dos autoespaços é $n$.
4. $m_g(\lambda) = m_a(\lambda)$ para todo $\lambda$.
5. $K^n = \bigoplus_\lambda E_\lambda$ — soma direta dos autoespaços.

### Construção

1. Calcule autovalores $\lambda_1, \ldots, \lambda_n$ (com multiplicidade).
2. Para cada $\lambda$, encontre base de $E_\lambda$.
3. Se total = $n$ vetores LI: $P$ = matriz com eles como colunas, $D = \text{diag}(\lambda_i)$.
4. Caso contrário: **não é diagonalizável** (use forma de Jordan).

### Aplicações imediatas

- **$A^k$**: $A^k = P D^k P^{-1}$, com $D^k = \text{diag}(\lambda_i^k)$.
- **$e^{At}$**: $e^{At} = P e^{Dt} P^{-1}$, com $e^{Dt} = \text{diag}(e^{\lambda_i t})$.
- **$f(A)$**: para qualquer $f$ analítica, $f(A) = P f(D) P^{-1}$.

### Casos garantidos

| Tipo | Diagonalizável? |
|---|---|
| Autovalores todos distintos | Sim (sempre, sobre $\mathbb{C}$) |
| Simétrica real | Sim, com $P$ ortogonal (espectral) |
| Hermitiana | Sim, com $P$ unitária |
| Normal ($AA^* = A^*A$) | Sim, com $P$ unitária |
| Triangular não-diagonal com diagonal repetida | Geralmente não |

<Porta nivel="5" titulo="Pra criança de 5 anos">

Imagina uma **caixa bagunçada**. Se você consegue **organizar tudo em colunas separadas** — uma cor por coluna — fica muito mais fácil mexer. **Diagonalizar** é exatamente isso: organizar a matriz em "cores" (autovalores) e "colunas" (autovetores).

Depois de organizada, fazer **muitas operações de uma vez** vira contas simples — só multiplicar cada cor por ela mesma várias vezes.

</Porta>

<Porta nivel="10" titulo="Pra criança de 10 anos">

Diagonalizar uma matriz é **trocar de óculos**. Você vê o mesmo objeto, mas em uma vista onde **fica simples**.

Na vista normal, $A$ tem números espalhados — multiplicar é trabalhoso. Na vista diagonal $D$, só tem números na diagonal — multiplicar é só elevar cada número.

A **fórmula mágica**: $A = PDP^{-1}$. $P$ é a "**troca de óculos**" — leva da base de autovetores pra base canônica.

**Por que serve?** Você quer calcular $A^{100}$ — multiplicar $A$ por ela mesma 100 vezes. Sem diagonalizar, é trabalhoso. Diagonalizado: $A^{100} = P D^{100} P^{-1}$, e $D^{100}$ é só elevar cada autovalor a 100. Conta direta!

</Porta>

<Porta nivel="15" titulo="Pra adolescente de 15 anos">

Diagonalização é a **transformação que revela** o que a matriz faz. Em vez de números misturados, você vê **direções de esticamento separadas**.

### Por que é poderoso?

Toda iteração linear $x_{k+1} = A x_k$ tem solução fechada via diagonalização: escreva $x_0 = \sum c_i v_i$ (combinação dos autovetores). Então:

$$x_k = A^k x_0 = \sum c_i \lambda_i^k v_i$$

Cada componente cresce/decai independente, com taxa $\lambda_i$. **Modos próprios** do sistema.

### Aplicação clássica: Fibonacci

$F_{n+1} = F_n + F_{n-1}$. Em matriz:
$$\begin{pmatrix} F_{n+1} \\ F_n \end{pmatrix} = \begin{pmatrix} 1 & 1 \\ 1 & 0 \end{pmatrix} \begin{pmatrix} F_n \\ F_{n-1} \end{pmatrix}$$

Autovalores $\phi = (1 + \sqrt 5)/2$ e $\psi = (1 - \sqrt 5)/2$. Diagonalizando:

$$F_n = \frac{\phi^n - \psi^n}{\sqrt 5}$$

— **fórmula fechada de Binet** — sem recursão!

### Quando NÃO diagonaliza?

- Multiplicidade geométrica $<$ algébrica.
- Exemplo clássico: $\begin{pmatrix} 2 & 1 \\ 0 & 2 \end{pmatrix}$ — autovalor 2 com $m_a = 2$, mas $m_g = 1$.

Nesse caso, usa-se **forma de Jordan**: blocos como $J = \begin{pmatrix} \lambda & 1 \\ 0 & \lambda \end{pmatrix}$.

### Ortogonalmente diagonalizável

Caso especial **muito útil**: $A$ simétrica → $A = QDQ^T$ com $Q$ ortogonal ($Q^T Q = I$). Inversão = transposição. **Teorema espectral**, próxima aula (116).

</Porta>

<Porta nivel="25" titulo="Pra estudante de engenharia">

Diagonalização transforma **álgebra acoplada em álgebra desacoplada**. É a operação base de toda análise modal em engenharia.

### Aplicação 1: sistemas LTI

$\dot x = Ax$, $x(0) = x_0$. Se $A = PDP^{-1}$:

$$x(t) = e^{At} x_0 = P e^{Dt} P^{-1} x_0$$

Em coordenadas $y = P^{-1} x$: $\dot y = D y$ — sistema **desacoplado**! Cada componente $y_i$ evolui independente como $y_i(t) = y_i(0) e^{\lambda_i t}$.

### Aplicação 2: análise modal mecânica

Sistema massa-mola: $M \ddot x + Kx = 0$. Generalizado: $K v = \lambda M v$. Modos próprios = autovetores. Frequências $\omega_i = \sqrt{\lambda_i / m_i}$.

Em estruturas (prédios, asas), análise modal identifica **modos críticos** (ressonância). Tacoma Narrows colapsou por excitação de modo torsional.

### Aplicação 3: cadeias de Markov

$\pi_{t+1} = \pi_t P$. Distribuição estacionária: autovetor de $\lambda = 1$. Se $P$ irredutível e aperiódica, $\pi_t \to \pi_\infty$ exponencialmente, com **taxa governada pelo segundo maior autovalor** ($\lambda_2$).

### Aplicação 4: potência matricial

Quer simular 30 anos de população com matriz Leslie? $x_{30} = A^{30} x_0$. Diagonalizando, $A^{30} = P D^{30} P^{-1}$ — uma multiplicação em vez de 30.

### Custo computacional

- Decomposição completa: $O(n^3)$ via QR algorithm.
- $A^k v$ via diagonalização: $O(n^3)$ uma vez + $O(n^2)$ por iteração.
- Sem diagonalizar: $O(n^3)$ por iteração.

Vantagem clara para iterações longas.

### Quando NÃO diagonalizar?

- $A$ esparsa, querendo apenas alguns modos: use Lanczos/Arnoldi.
- $A$ próxima de defectiva: $P$ mal-condicionada, instável.
- Em ML, frequentemente trabalha-se com $A^T A$ (sempre simétrica → ortodiagonalização) via SVD.

### Aplicação 5: Google PageRank

Iteração $\pi \leftarrow \pi P$. Power iteration converge ao autovetor de $\lambda = 1$. Toda a indexação web original (1998) era esta iteração.

</Porta>

<Porta nivel="40" titulo="Pra profissional sênior">

### Generalização: Jordan canonical form

Quando $A$ não é diagonalizável: $A = PJP^{-1}$ com $J$ bloco-diagonal:

$$J = \begin{pmatrix} J_{n_1}(\lambda_1) & & \\ & \ddots & \\ & & J_{n_k}(\lambda_k) \end{pmatrix}, \quad J_n(\lambda) = \lambda I_n + N$$

com $N$ shift nilpotente. Aplicações de funções: $f(J_n(\lambda))$ via série de Taylor truncada (porque $N^n = 0$).

Em prática numérica, **Jordan é mal-condicionada** — usa-se Schur ($A = QTQ^*$, $Q$ unitária, $T$ triangular superior) que é estável.

### Diagonalização simultânea

Conjunto $\{A_i\}$ é simultaneamente diagonalizável $\iff $ todos comutam: $A_i A_j = A_j A_i$ para todo par.

Em mecânica quântica, **observáveis compatíveis** ↔ comutam ↔ podem ser medidos simultaneamente. Princípio de incerteza nasce daqui.

### Pseudoespectro

Para matrizes não-normais, autovalores podem ser **enganosos** — pequena perturbação muda muito. Definição:

$$\sigma_\epsilon(A) = \{z \in \mathbb{C} : \|(zI - A)^{-1}\| > 1/\epsilon\}$$

Trefethen mostrou que pseudoespectro explica fenômenos de transient growth (Couette flow, redes neurais com atividade explosiva).

### Conexão com BS (Aula 119)

**Greeks** de Black-Scholes em portfólios → matrizes de sensibilidade $\partial^2 V / \partial S_i \partial S_j$. Diagonalização revela direções principais de risco — esta é a base de **risk attribution** moderna (Litterman, Risk Metrics).

### Aplicações modernas

- **Spectral methods em PDEs**: diagonaliza operador via base ortonormal (Fourier, Chebyshev). Convergência espectral (super-algébrica).
- **Koopman operator**: extensão linear de dinâmica não-linear → diagonalização infinita-dim. DMD (Schmid 2010) é versão data-driven.
- **Neural network spectral analysis**: estabilidade de treinamento via espectro dos jacobianos.
- **Matrix exponential em deep learning**: skip connections + escolha de arquitetura mantém autovalores próximos de 1 (evita explosion/vanishing).

</Porta>

<Porta nivel="pratica" titulo="Aplicação prática">

### Receita de bolo

1. Calcule autovalores $\lambda_1, \ldots, \lambda_n$.
2. Para cada $\lambda$, encontre base de $E_\lambda = \ker(A - \lambda I)$.
3. Some as dimensões — se chegar em $n$, é diagonalizável. Senão, não é.
4. Monte $P = [v_1 | v_2 | \cdots | v_n]$ e $D = \text{diag}(\lambda_i)$.
5. Verifique: $AP = PD$ (ou equivalentemente $A = PDP^{-1}$).

### Em código

```python
import numpy as np
from scipy.linalg import expm

A = np.array([[4, 1], [2, 3]])
vals, P = np.linalg.eig(A)
D = np.diag(vals)
print("Erro decomposição:", np.linalg.norm(A - P @ D @ np.linalg.inv(P)))

# A^10 via diagonalização
A10 = P @ np.diag(vals**10) @ np.linalg.inv(P)

# e^{At}
def exp_diagonalizable(A, t):
    vals, P = np.linalg.eig(A)
    return P @ np.diag(np.exp(vals*t)) @ np.linalg.inv(P)
```

### Atalho importante

Se $A$ é **simétrica real**, sempre é diagonalizável com $P$ ortogonal. Use `np.linalg.eigh` (mais rápido e estável).

### Quando preocupar?

- **Multiplicidade $m_g < m_a$**: não diagonaliza. Use Jordan ou trabalhe sobre $\mathbb{C}$.
- **Autovalores complexos**: aparecem em rotações. Em geral OK, mas $P$ é complexa.
- **Mal-condicionamento**: $\kappa(P)$ alto → erros amplificados. Para simétrica, $P$ ortogonal → $\kappa = 1$ (perfeito).
