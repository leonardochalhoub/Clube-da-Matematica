### Mapa do trimestre

| Aula | Tópico | Equação chave |
|---|---|---|
| 102 | IC média | $\bar X \pm z_{\alpha/2}\sigma/\sqrt n$ |
| 103 | Teste hipótese | $p \leq \alpha \Rightarrow$ rejeita $H_0$ |
| 104 | Teste z e t | $T = (\bar X - \mu_0)/(s/\sqrt n)$ |
| 105 | Regressão simples | $\hat\beta_1 = S_{xy}/S_{xx}$ |
| 106 | Regressão múltipla | $\hat\beta = (X^TX)^{-1}X^Ty$ |
| 107 | ANOVA | $F = MS_B/MS_W$ |
| 108 | Qui-quadrado | $\chi^2 = \sum (O-E)^2/E$ |
| 109 | Bayes | $P(\theta\|D) \propto P(D\|\theta)P(\theta)$ |

### Os 3 pilares da inferência

1. **Estimação** com IC: faixa plausível para parâmetro.
2. **Teste** de hipótese: decidir entre $H_0$ e $H_1$.
3. **Modelagem** preditiva: regressão, ANOVA.

### Sob qual condição cada teste?

| Cenário | Teste |
|---|---|
| 1 média, $\sigma$ conhecido | Teste z |
| 1 média, $\sigma$ desconhecido | Teste t |
| 2 médias independentes | Welch's t |
| Pareado | t pareado |
| $k$ médias | ANOVA + Tukey |
| 2 proporções | z para proporções |
| Tabela de contingência | $\chi^2$ |
| Aderência | $\chi^2$ goodness of fit |
| Predição linear | Regressão |
| Crença atualizada | Bayes |

### Síntese: estatística inferencial

**Ponto de vista frequentista** (102-108):

- Parâmetros são fixos, dados são aleatórios.
- IC: cobertura sobre amostras hipotéticas.
- $p$-valor: probabilidade dos dados sob $H_0$.

**Ponto de vista bayesiano** (109):

- Parâmetros são variáveis aleatórias.
- Posterior: probabilidade direta sobre $\theta$.
- Atualização: prior + likelihood = posterior.

### Conexões internas

- **Regressão = ANOVA = teste t**: caso particular do modelo linear geral.
- **$\chi^2 \approx \sum Z_i^2$**: relação com normal.
- **MAP com prior uniforme = MLE**: bayesiano e frequentista coincidem.
- **F = T²** quando $k=2$ na ANOVA.

### Erros recorrentes em todos os métodos

1. **Múltiplas comparações sem correção**.
2. **Suposições não verificadas** (normalidade, independência).
3. **Confundir associação com causa**.
4. **Significância estatística vs prática**.
5. **$p$-hacking**.

### Boas práticas

- **Pré-registrar** hipóteses.
- **Power analysis** antes de coletar.
- **Reportar IC, $p$, tamanho de efeito**.
- **Diagnóstico de modelo**.
- **Replicação** quando possível.

### Generalização: tudo é modelo linear (quase)

Regressão, ANOVA, teste t, $\chi^2$ (em forma log-linear) — todos são casos particulares do **GLM** (modelo linear generalizado):

$$g(E[Y]) = X\beta$$

| Método | Likelihood | Link |
|---|---|---|
| Regressão linear | Normal | identidade |
| ANOVA | Normal | identidade |
| Logística | Bernoulli | logit |
| Poisson | Poisson | log |
| Multinomial | Multinomial | logit multivariado |

Mestre do GLM = mestre da inferência paramétrica clássica.

### Resampling: paradigma alternativo

- **Bootstrap**: para IC, EP de qualquer estatística.
- **Permutação**: para $p$-valor sem suposição paramétrica.
- **Cross-validation**: para validação preditiva.

Esses superam paramétrico em muitos cenários reais.

### Frequentista vs Bayesiano: complementares

| Pergunta | Bom para |
|---|---|
| "Que dados são improváveis sob $H_0$?" | Frequentista |
| "Qual probabilidade de $H$ dado dados?" | Bayesiano |
| "Decisão sob risco" | Bayesiano |
| "Long-run error rates" | Frequentista |

Use ambos, não rivais.

### Power analysis (subutilizada!)

Antes de qualquer coleta:

```python
from statsmodels.stats.power import TTestPower
analysis = TTestPower()
n = analysis.solve_power(effect_size=0.5, power=0.8, alpha=0.05)
```

Sem isso, experimentos morrem antes de nascer.

### Reprodutibilidade

- **Pré-registro** (OSF, AsPredicted).
- **Código aberto**.
- **Dados públicos** quando ético.
- **Replicação** independente.

### Workshop integrado

Dado um problema real (ex.: A/B testing em e-commerce):

1. **Define** métrica primária e secundária.
2. **Power analysis**: $n$ necessário.
3. **Aleatoriza**: design experimental.
4. **Coleta**: até $n$ alvo.
5. **Analisa**: teste z para proporções + IC.
6. **Reporta**: completo, sem $p$-hacking.

### Implementação master

```python
import scipy.stats as st
import statsmodels.api as sm
import numpy as np

# IC
mean, se = data.mean(), st.sem(data)
ci = st.t.interval(0.95, len(data)-1, loc=mean, scale=se)

# Teste t
t, p = st.ttest_ind(A, B, equal_var=False)

# Regressão
X = sm.add_constant(x); model = sm.OLS(y, X).fit()

# ANOVA
F, p = st.f_oneway(g1, g2, g3)

# Qui-quadrado
chi2, p, dof, exp = st.chi2_contingency(table)
```

### Filosofia: o que é inferência?

- **Frequentista** (Neyman-Pearson): controlar long-run error rates.
- **Bayesiano** (Jeffreys, Lindley): atualizar crenças coerentemente.
- **Likelihoodista** (Fisher, Edwards): comparar verossimilhanças.
- **Causal** (Pearl, Rubin): estimar contrafactuais.

### Crise de replicação

2015: Open Science Collaboration — 36% dos resultados em psicologia replicam. Causas:

1. $p$-hacking, HARKing.
2. Power analysis ignorada.
3. Múltiplas comparações.
4. Dados não compartilhados.
5. Vieses publicação.

Soluções: pré-registro, replicação direta, código aberto, métricas além de $p < 0{,}05$.

### Modern statistics

- **Causal inference**: DAGs, do-calculus, IV, RDD, DiD.
- **Machine learning**: predição vs inferência. Bias-variance tradeoff.
- **High-dimensional**: $p \gg n$. Lasso, knockoffs.
- **Bayesian computation**: HMC, VI, ABC.
- **Selective inference**: post-selection valid p-values.

### Software stack moderno

- **R**: tidyverse, brms, lme4, marginaleffects, easystats.
- **Python**: scipy, statsmodels, pymc, sklearn, lifelines.
- **Stan**: HMC para qualquer modelo.
- **JAX**: differentiable programming.

### Etica em estatística

- **p-hacking** = fraude leve.
- **Dredge** sem disclosure = fraude.
- **Não reportar análises pré-especificadas** = fraude.

Veja ASA Statement on $p$-values (2016).

### Caso 1: A/B testing

Versão A vs B do checkout. Métrica: conversão. $n_A = n_B = 5000$, $s_A = 250$, $s_B = 290$.

1. **Frequentista**: teste z para proporções.
2. **Bayesiano**: posterior $\text{Beta}(s+1, n-s+1)$, $P(\theta_B > \theta_A)$ via simulação.
3. **Reporte**: ambos.

### Caso 2: ensaio clínico

Droga vs placebo, $n=200$ cada, redução de pressão (mmHg). $\bar D_{\text{droga}} = 8 \pm 4$, $\bar D_{\text{placebo}} = 2 \pm 4$.

1. Welch t-test.
2. Tamanho de efeito Cohen's d.
3. IC 95% para diferença.

### Caso 3: regressão preditiva

Preço de imóvel vs $m^2$, quartos, idade. $n=200$ casas.

1. Ajuste OLS múltipla.
2. VIF para colinearidade.
3. Diagnóstico de resíduos.
4. $R^2_{\text{adj}}$.
5. Predição com IC.

### Caso 4: ANOVA com post-hoc

4 fertilizantes, 25 plantas cada. Rendimento médio.

1. Levene + Shapiro-Wilk.
2. ANOVA.
3. Tukey HSD.
4. $\eta^2$.

### Caso 5: tabela de contingência

Marca $\times$ região: 5x4 tabela. Independência?

1. $\chi^2$ test.
2. Cramér's V.
3. Resíduos padronizados.

### Caso 6: bayesiano end-to-end

Conversão de feature toggle: prior $\text{Beta}(2, 8)$, observa 30/100.

1. Posterior.
2. IC credível 95%.
3. $P(\theta > 0{,}25 | D)$ via simulação.

### Workflow profissional

```python
def analyze_ab_test(s_A, n_A, s_B, n_B):
    from scipy import stats
    import numpy as np

    # Frequentista
    p_A, p_B = s_A/n_A, s_B/n_B
    pooled = (s_A + s_B) / (n_A + n_B)
    se = np.sqrt(pooled * (1-pooled) * (1/n_A + 1/n_B))
    z = (p_B - p_A) / se
    p_value = 2 * (1 - stats.norm.cdf(abs(z)))

    # Bayesiano
    A = np.random.beta(s_A + 1, n_A - s_A + 1, 100000)
    B = np.random.beta(s_B + 1, n_B - s_B + 1, 100000)
    prob_B_better = (B > A).mean()

    return dict(p_value=p_value, prob_B_better=prob_B_better, lift=p_B/p_A - 1)
```
