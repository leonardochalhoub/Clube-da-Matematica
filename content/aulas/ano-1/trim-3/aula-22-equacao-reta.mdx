<Porta nivel="formal" titulo="Formas da equação da reta">

### Forma reduzida

$y = mx + n$, onde $m$ é o **coeficiente angular** (inclinação) e $n$ é o **coeficiente linear** (intercepto em $y$).

### Forma geral

$Ax + By + C = 0$, com $(A, B) \neq (0, 0)$. Para retas não-verticais ($B \neq 0$): $m = -A/B$ e $n = -C/B$.

### Forma ponto-inclinação

Reta passando por $(x_0, y_0)$ com inclinação $m$:
$$y - y_0 = m(x - x_0)$$

### Forma paramétrica

Reta com vetor diretor $\vec u = (a, b)$ passando por $(x_0, y_0)$:
$$\begin{cases} x = x_0 + at \\ y = y_0 + bt \end{cases}, \quad t \in \mathbb{R}$$

### Equação por dois pontos

Reta passando por $(x_1, y_1)$ e $(x_2, y_2)$:
$$\frac{y - y_1}{x - x_1} = \frac{y_2 - y_1}{x_2 - x_1}$$

### Coeficiente angular por dois pontos

$$m = \frac{y_2 - y_1}{x_2 - x_1} = \frac{\Delta y}{\Delta x} = \tan\alpha$$

onde $\alpha$ é o ângulo da reta com o eixo $x$.

### Tabela das 5 formas

| Forma | Equação | Quando usar |
|---|---|---|
| Reduzida | $y = mx + n$ | gráfico, $y$ explícito |
| Geral | $Ax + By + C = 0$ | retas verticais, simetria |
| Ponto-inclinação | $y - y_0 = m(x - x_0)$ | ponto + inclinação dados |
| Paramétrica | $(x_0 + at, y_0 + bt)$ | trajetória, animação |
| Segmentária | $x/p + y/q = 1$ | interceptos $(p, 0), (0, q)$ |

</Porta>

<Porta nivel="5" titulo="Pra criança de 5 anos">

Imagina uma **rampinha**. Ela sobe ou desce, mas sempre **na mesma inclinação** — não tem curva. Isso é uma **reta**.

Pra descrever a rampa, basta saber duas coisas: **onde ela começa** (intercepto) e **quão íngreme é** (inclinação). Pronto, dá pra adivinhar onde a rampa está em qualquer lugar.

</Porta>

<Porta nivel="10" titulo="Pra criança de 10 anos">

Reta no plano cartesiano é descrita por $y = mx + n$. O **$m$** é a inclinação (o quanto sobe pra cada passo de $x$). O **$n$** é onde a reta cruza o eixo vertical.

**Exemplo concreto**: $y = 2x + 3$. Quando $x = 0$, $y = 3$ — começa em 3 no eixo $y$. Quando $x$ aumenta 1, $y$ aumenta 2 — sobe rápido.

**Aplicação real**: tarifa de táxi. Bandeirada R\$ 5 + R\$ 2 por km. Equação: $y = 2x + 5$. Onde $y$ é o preço e $x$ é a distância. Reta perfeita.

**Truque pra achar a reta entre 2 pontos**:
- Calcula $m = \frac{y_2 - y_1}{x_2 - x_1}$.
- Substitui um ponto em $y = mx + n$ pra achar $n$.

</Porta>

<Porta nivel="15" titulo="Pra adolescente de 15 anos">

A equação da reta é o que conecta a álgebra à geometria. Cada reta no plano tem uma equação $y = mx + n$ que descreve **todos os pontos** dela.

- $m$ é a **inclinação** (quão "subindo" ou "descendo" está a reta). $m = \tan\alpha$ onde $\alpha$ é o ângulo com o eixo $x$.
- $n$ é onde a reta cruza o eixo $y$.

Quando você tem **dois pontos** $(x_1, y_1)$ e $(x_2, y_2)$, calcula a inclinação por:

$$m = \frac{y_2 - y_1}{x_2 - x_1}$$

E depois descobre $n$ substituindo um dos pontos.

**Forma geral** $Ax + By + C = 0$ é mais "simétrica" — trata $x$ e $y$ igualmente. Útil quando a reta é vertical ($x = $ constante), o que a forma reduzida não consegue expressar.

**Conexão Aula 03**: você já viu função afim $f(x) = ax + b$. Reta é o gráfico dessa função, com $a = m$ e $b = n$. Mesma coisa, língua diferente.

**Exemplo numérico completo**: reta por $(2, 3)$ e $(5, 9)$. $m = (9-3)/(5-2) = 2$. Substitui $(2, 3)$: $3 = 2 \cdot 2 + n \Rightarrow n = -1$. Equação: $y = 2x - 1$.

</Porta>

<Porta nivel="25" titulo="Pra estudante de engenharia">

Equação da reta é base de toda análise linear. Em $\mathbb{R}^n$, o equivalente é o **hiperplano** $\sum a_i x_i = b$.

### Aplicações imediatas

- **Programação linear**: maximizar $c^Tx$ sujeito a $Ax \leq b$ — restrições lineares (semi-espaços limitados por hiperplanos). Simplex viaja por vértices.
- **Regressão linear simples**: $\hat y = \hat m x + \hat n$ ajustada via mínimos quadrados. Estimativa OLS:
$$\hat m = \frac{\sum (x_i - \bar x)(y_i - \bar y)}{\sum (x_i - \bar x)^2}$$
- **Cinemática**: posição em movimento retilíneo é $s(t) = s_0 + vt$ — reta no plano $(t, s)$.
- **Computação gráfica**: rasterização de retas (algoritmo de Bresenham, 1965 — IBM).
- **SVMs**: classificador encontra hiperplano de maior margem entre classes.

### Distância ponto-reta

Para reta $Ax + By + C = 0$ e ponto $P_0 = (x_0, y_0)$:
$$d(P_0, \text{reta}) = \frac{|A x_0 + B y_0 + C|}{\sqrt{A^2 + B^2}}$$

Demonstração via projeção vetorial — o vetor $(A, B)$ é normal à reta (perpendicular).

### Forma normal de Hesse

$$x \cos\theta + y \sin\theta = p$$

onde $p \geq 0$ é a distância da origem à reta e $\theta$ o ângulo da normal. Útil em geometria computacional.

### Conexão com cálculo

A reta tangente a uma curva $y = f(x)$ no ponto $(a, f(a))$ é:
$$y = f(a) + f'(a)(x - a)$$

— a **linearização** local. Newton-Raphson, séries de Taylor, otimização gradient descent, todos baseados nesta ideia.

</Porta>

<Porta nivel="40" titulo="Pra profissional sênior">

Reta = subespaço afim 1-dimensional de $\mathbb{R}^n$. Toda a geometria projetiva moderna começa daqui.

### Visão projetiva

Em coordenadas homogêneas $(X, Y, W)$ com $x = X/W$, $y = Y/W$, a reta vira:
$$AX + BY + CW = 0$$

— **forma linear pura**. Pontos no infinito ($W = 0$) ficam acessíveis. Duas retas paralelas no afim se cruzam num ponto no infinito no projetivo. Esta é a base de **OpenGL**, **PBR** e toda transformação 3D moderna (rotações, translações, projeções perspectivas).

### Dualidade

No plano projetivo, **pontos e retas são duais**: cada teorema sobre pontos tem versão sobre retas trocando "passa por" ↔ "contém". Pascal ↔ Brianchon. Crucial em geometria algébrica.

### Aplicações em ML / quant

- **Linear regression** generaliza pra $\hat y = X\beta$. Sistema normal $X^TX\beta = X^Ty$ resolve via QR ou SVD.
- **PCA**: projeções em hiperplanos que minimizam variância residual.
- **Quant finance**: Capital Asset Pricing Model (CAPM): $E[R_i] = R_f + \beta_i (E[R_m] - R_f)$ — reta no plano $(\beta, E[R])$. Markowitz frontier é envoltória; CAPM é tangente. Fama-French estende para hiperplano em 3+ dimensões.
- **Gradient descent**: a cada passo, aproxima função por reta tangente e move-se na direção de descida. Toda otimização first-order é "reta com inclinação $-\nabla f$".

### SymPy

```python
import sympy as sp
from sympy import Line, Point
L = Line(Point(2, 3), Point(5, 9))
print(L.equation())     # 2x - y - 1
print(L.slope)          # 2
print(L.distance(Point(0, 0)))  # 1/sqrt(5)
```

### Numérico

Em IEEE 754, retas quase-verticais ($|m|$ enorme) sofrem cancelamento catastrófico. Use forma geral $Ax + By + C = 0$ com $(A, B)$ unitário pra estabilidade. Geometria computacional robusta (CGAL, predicate of Shewchuk) usa aritmética exata pra evitar erros.

<Porta nivel="pratica" titulo="Aplicação prática">

**Receita: como achar equação de reta em qualquer cenário.**

| Dado | Estratégia |
|---|---|
| 2 pontos | $m = \Delta y / \Delta x$, depois substitui |
| 1 ponto + inclinação | ponto-inclinação direto |
| 1 ponto + reta paralela | mesmo $m$, novo $n$ |
| 1 ponto + reta perpendicular | $m \to -1/m$ |
| Forma geral $\to$ reduzida | $y = -A/B \cdot x - C/B$ |
| Reduzida $\to$ geral | $mx - y + n = 0$ |

### Workflow Wolfram / SymPy

```python
import sympy as sp
x, y, t = sp.symbols('x y t')

# Reta por dois pontos
P, Q = sp.Point(1, 2), sp.Point(4, 8)
L = sp.Line(P, Q)
print(L.equation())   # forma geral

# Forma paramétrica
print(L.arbitrary_point(t))
```

### Quando preocupar?

- **Reta vertical** ($x = c$) não tem $m$ definido. Use forma geral.
- **Pontos coincidentes** ($x_1 = x_2$, $y_1 = y_2$): não definem reta única. Erro de input.
- **Inclinação infinita numericamente**: cancelamento — prefira $\Delta x \cdot (y - y_0) = \Delta y \cdot (x - x_0)$.
