<Porta nivel="formal" titulo="Definição e tipos">

<Definicao titulo="Matriz">
Matriz $m \times n$ sobre $\mathbb{R}$ é uma função $A: \{1, \ldots, m\} \times \{1, \ldots, n\} \to \mathbb{R}$. Notação $A = (a_{ij})$ ou $A_{m \times n}$.

Igualdade: $A = B \iff$ mesma dimensão e $a_{ij} = b_{ij}$ para todos $i, j$.
</Definicao>

### Tipos especiais

| Tipo | Definição |
|---|---|
| Quadrada | $m = n$ |
| Linha | $1 \times n$ |
| Coluna | $m \times 1$ |
| Diagonal | quadrada com $a_{ij} = 0$ para $i \neq j$ |
| Identidade $I_n$ | diagonal com $a_{ii} = 1$ |
| Nula $O$ | todos os elementos nulos |
| Triangular superior | $a_{ij} = 0$ para $i > j$ |
| Triangular inferior | $a_{ij} = 0$ para $i < j$ |
| Simétrica | $A^T = A$, ou seja $a_{ij} = a_{ji}$ |
| Antissimétrica | $A^T = -A$, ou seja $a_{ij} = -a_{ji}$ |
| Escalar | diagonal com $a_{ii} = k$ constante |

### Diagonal de matriz quadrada

A **diagonal principal** é $\{a_{ii}\}$. **Traço**: $\text{tr}(A) = \sum_{i=1}^n a_{ii}$.

### Lei de formação

Frequentemente se define $A$ via fórmula $a_{ij} = f(i, j)$. Exemplos:
- $a_{ij} = i + j$
- $a_{ij} = (-1)^{i+j}$
- $a_{ij} = \delta_{ij}$ (delta de Kronecker — gera identidade)

</Porta>

<Porta nivel="5" titulo="Pra criança de 5 anos">

Imagina uma **caixa de ovos**. Tem fileiras e colunas. Cada espaço guarda um ovo (que é um número).

Uma **matriz** é só isso: uma caixa quadradona ou retangular cheia de números, organizados certinho — cada um no seu lugar. Pra dizer onde um número está, você fala a fileira e a coluna: "linha 2, coluna 3" — pronto, achou.

</Porta>

<Porta nivel="10" titulo="Pra criança de 10 anos">

Você já viu **tabelas** na sua aula. Uma tabela de notas: cada aluno é uma linha, cada matéria é uma coluna. **Matriz é exatamente isso** — uma tabela de números.

Exemplos do dia a dia:
- **Boletim**: 30 alunos × 5 matérias = matriz 30×5.
- **Tela do computador**: cada pixel tem um valor de cor. Tela de 1920×1080 é uma matriz gigante.
- **Mapa de distâncias**: tabela cidade × cidade. Linha São Paulo, coluna Rio = 430 km.

Cada número tem um **endereço**: $a_{ij}$ é "linha $i$, coluna $j$". O $i$ vem primeiro, sempre.

Tipos especiais que aparecem muito:
- **Quadrada**: número de linhas igual ao de colunas.
- **Identidade**: igual ao número 1 — só tem 1 na diagonal e 0 no resto.
- **Nula**: só tem zeros.

</Porta>

<Porta nivel="15" titulo="Pra adolescente de 15 anos">

Matriz é uma **tabela** de números organizada em linhas e colunas:

$$A = \begin{pmatrix} 1 & 2 & 3 \\ 4 & 5 & 6 \end{pmatrix}$$

Esta é uma matriz $2 \times 3$ (2 linhas, 3 colunas). Cada número tem **endereço**: $a_{12} = 2$ (linha 1, coluna 2). $a_{23} = 6$ (linha 2, coluna 3).

**Para que serve?** Para guardar e organizar dados, e principalmente para **fazer contas em massa**:
- Notas dos alunos × disciplinas: matriz aluno-disciplina.
- Distâncias entre cidades: matriz simétrica.
- Pixels de uma imagem: matriz de intensidade.
- Coeficientes de um sistema linear: matriz de coeficientes.
- Pesos de uma rede neural: vetores e matrizes (toneladas).

**Igualdade**: duas matrizes são iguais se tiverem **mesma dimensão** E **todos os elementos correspondentes iguais**. Não dá pra comparar $2 \times 3$ com $3 \times 2$.

**Tipos importantes**:
- **Quadrada**: tem o mesmo número de linhas e colunas.
- **Identidade** $I_n$: tem 1's na diagonal e 0 em toda a outra parte. Funciona como o "1" da multiplicação ($IA = A$).
- **Diagonal**: só tem zeros fora da diagonal principal.
- **Simétrica**: $A^T = A$ — espelho perfeito pela diagonal.

**Lei de formação**: dá pra construir matriz a partir de uma fórmula. Se $a_{ij} = i + j$, então $a_{12} = 3$, $a_{21} = 3$, $a_{33} = 6$, etc.

</Porta>

<Porta nivel="25" titulo="Pra estudante de engenharia">

Matriz é o **objeto fundamental** da computação científica e do ML. Em Python/NumPy, "tudo é array". Em MATLAB, "tudo é matriz". O termo vem de **Cayley (1858)**, mas a estrutura já estava implícita em Lagrange e Gauss.

### Visão como aplicação linear

Uma matriz $A_{m \times n}$ representa uma aplicação linear $T: \mathbb{R}^n \to \mathbb{R}^m$ via $T(\mathbf x) = A\mathbf x$. As **colunas** são imagens dos vetores da base canônica. Esta é a interpretação que torna matrizes onipresentes.

### Estruturas matriciais comuns

- **Esparsa**: maioria dos elementos é zero. Formatos CSR/CSC armazenam só não-zeros. Crucial em FEM, redes, NLP, grafos com milhões de nós.
- **Banda**: zeros fora de uma faixa central. Útil em sistemas tridiagonais (calor, Crank-Nicolson, splines cúbicas).
- **Densa**: tudo armazenado. Padrão em ML, álgebra linear pura, BLAS-3.
- **Block-diagonal**: blocos quadrados na diagonal — sistemas desacoplados.
- **Toeplitz**: $a_{ij}$ depende só de $i - j$ — convoluções, processamento de sinais.

### Aplicações cross-domínio

- **Sistemas de controle**: estado $\dot{\mathbf{x}} = A\mathbf{x} + B\mathbf{u}$, $\mathbf y = C\mathbf x$ — A é a "dinâmica".
- **Estruturas (FEM)**: matriz de rigidez global $K$ é esparsa, simétrica, definida positiva.
- **Imagem digital**: cada canal RGB é matriz $H \times W$ de intensidades $\{0, \ldots, 255\}$.
- **Redes neurais**: cada camada densa é $\mathbf y = \sigma(W\mathbf x + \mathbf b)$.
- **Quântica**: estados são vetores, observáveis são matrizes hermitianas $A = A^*$.
- **Estatística**: matriz de covariância $\Sigma$ é simétrica positiva semidefinida.
- **Grafos**: matriz de adjacência $A_{ij} = 1$ se há aresta — autovalores codificam estrutura.

### Espaço vetorial $M_{m \times n}(\mathbb R)$

O conjunto de matrizes $m \times n$ é um espaço vetorial de **dimensão** $mn$ — base canônica $\{E_{ij}\}$ onde $E_{ij}$ tem 1 na posição $(i,j)$ e 0 no resto.

</Porta>

<Porta nivel="40" titulo="Pra profissional sênior">

Matrizes são instâncias finitas de **operadores lineares** em espaços vetoriais. A teoria geral vive em $\text{Hom}(V, W)$ para $V, W$ espaços vetoriais — fixadas bases, $\text{Hom}(V, W) \cong M_{m \times n}$.

### Visão estrutural

- **$M_n(\mathbb R)$ é uma álgebra associativa não-comutativa** com unidade $I_n$.
- **Decomposições espectrais**: SVD ($A = U\Sigma V^T$), autovalores ($A = PDP^{-1}$ se diagonalizável), Schur, Jordan — cada uma revela aspecto distinto.
- **Grupos de matrizes** ($GL_n$, $SO_n$, $U_n$, $Sp_{2n}$) são objetos centrais em geometria, física e teoria de representação.

### Numérico moderno

- **BLAS/LAPACK**: GEMM (multiplicação) é o kernel mais otimizado da computação. $O(n^3)$ teórico, mas memória/cache dominam — flops "grátis" em GPU.
- **Autodiff e ML**: bibliotecas (PyTorch, JAX) tratam tensores como matrizes generalizadas. Backprop é regra da cadeia em grafos de operações matriciais.
- **Aleatorização**: random projections (Johnson-Lindenstrauss), sketching, randomized SVD reduzem complexidade em big data.

### Quant/Finance

- **Matriz de covariância** $\Sigma$ de retornos é base de Markowitz. Estimação robusta (shrinkage, Ledoit-Wolf) é problema central.
- **Matrizes de transição** em cadeias de Markov modelam ratings de crédito (S&P, Moody's).
- **Cholesky** $\Sigma = LL^T$ gera amostras Gaussianas correlacionadas em Monte Carlo.

### Generalizações

- **Tensores** ($T_{ijk\ldots}$): generalizam matriz a múltiplos índices. Decomposições (CP, Tucker) são análogos do SVD.
- **Operadores em dim infinita**: integrais de Fredholm, operadores compactos — análogos contínuos.
- **Matrizes sobre corpos finitos**: códigos corretores, criptografia (Niederreiter, McEliece), combinatória.

<Porta nivel="pratica" titulo="Aplicação prática">

**Workflow para trabalhar com matrizes na prática.**

1. **Identifique a estrutura**: dimensão? simétrica? esparsa? triangular? Estrutura economiza memória e flops.
2. **Escolha a representação**: NumPy array (denso), `scipy.sparse` (esparso), `pandas.DataFrame` (com rótulos), `xarray` (multi-dim com nomes).
3. **Vetorize**: nunca faça loops `for` para operações elemento-a-elemento — use broadcasting.

### NumPy — receita

```python
import numpy as np

# Construção
A = np.array([[1, 2, 3], [4, 5, 6]])     # 2x3 explícita
I = np.eye(3)                             # identidade 3x3
Z = np.zeros((4, 5))                      # nula 4x5
D = np.diag([2, -1, 5])                   # diagonal

# Lei de formação
i, j = np.indices((3, 3))
A_lei = i + j                             # a_ij = i + j

# Acesso
print(A[0, 2])      # elemento linha 0 coluna 2 (indexação 0-based!)
print(A.shape)      # (2, 3)
print(A.T)          # transposta
print(np.trace(np.eye(5)))   # traço
```

### SymPy — simbólico

```python
import sympy as sp
A = sp.Matrix([[1, 2], [3, 4]])
print(A.shape, A.T, A.trace())
```

### Conferência de simetria

```python
def is_symmetric(A, tol=1e-9):
    return np.allclose(A, A.T, atol=tol)
```

### Pegadinhas comuns

- **Indexação 0-based** em Python/NumPy vs **1-based** na matemática. Cuidado ao traduzir fórmulas.
- **Linha vs coluna**: `A[i]` é a linha $i$. Para coluna use `A[:, j]`.
- **Cópia vs view**: `B = A` não copia! Use `B = A.copy()`.
- **`np.matrix` está deprecated**. Use `np.ndarray` (`@` para multiplicação matricial).
