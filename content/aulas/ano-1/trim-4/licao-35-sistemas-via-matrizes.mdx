<Porta nivel="formal" titulo="Métodos de resolução">

### Forma matricial

$\begin{cases} a_1 x + b_1 y + c_1 z = d_1 \\ a_2 x + b_2 y + c_2 z = d_2 \\ a_3 x + b_3 y + c_3 z = d_3 \end{cases}$ ⟺ $A\mathbf{x} = \mathbf{b}$ com $A_{3 \times 3}$, $\mathbf{x}, \mathbf{b} \in \mathbb{R}^3$.

### Método 1 — Escalonamento de Gauss

Operações elementares (não alteram solução):
1. Trocar duas linhas.
2. Multiplicar uma linha por escalar não-nulo.
3. Somar múltiplo de uma linha em outra.

Objetivo: triangularizar $[A | \mathbf{b}]$ até forma escalonada. Depois retrosubstituição.

### Método 2 — Cramer

Para $A\mathbf{x} = \mathbf{b}$ com $\det A \neq 0$:
$$x_i = \frac{\det A_i}{\det A}$$

onde $A_i$ é $A$ com a $i$-ésima coluna substituída por $\mathbf{b}$.

### Método 3 — Inversa

$\mathbf{x} = A^{-1} \mathbf{b}$. Pode-se calcular $A^{-1}$ via $[A | I] \to [I | A^{-1}]$ por escalonamento.

### Quando usar cada

| Método | Custo | Quando usar |
|---|---|---|
| Cramer | $O(n^4)$ | $n \leq 3$, didático |
| Gauss | $O(n^3)$ | padrão prático |
| Inversa | $O(n^3)$ + $O(n^2)$/sistema | múltiplos $\mathbf b$ com mesmo $A$ |
| LU | $O(n^3)$ + $O(n^2)$/sistema | múltiplos sistemas, melhor que inversa |

### Classificação de sistemas

- **Determinado (SPD)**: solução única ($\det A \neq 0$, ou $\text{rank}(A) = \text{rank}([A|\mathbf b]) = n$).
- **Indeterminado (SPI)**: infinitas soluções ($\text{rank}(A) = \text{rank}([A|\mathbf b]) < n$).
- **Impossível (SI)**: sem solução ($\text{rank}(A) < \text{rank}([A|\mathbf b])$).

### Teorema de Rouché-Capelli

$A\mathbf x = \mathbf b$ tem solução ⟺ $\text{rank}(A) = \text{rank}([A|\mathbf b])$. Solução é única se ambos = número de incógnitas.

<Porta nivel="5" titulo="Pra criança de 5 anos">

Sistema é **um monte de equações ao mesmo tempo**. Tipo: "tenho 3 maçãs e 2 peras, total 13 reais. Tenho 1 maçã e 4 peras, total 16 reais. Quanto custa cada uma?"

Pra resolver, você **junta** as equações de um jeito esperto pra **eliminar** uma das frutas, descobre o preço da outra, e volta. É um quebra-cabeça com receita certa.

</Porta>

<Porta nivel="10" titulo="Pra criança de 10 anos">

**Sistema linear** = várias equações com várias incógnitas precisando ser verdadeiras ao mesmo tempo.

Exemplo:
$$\begin{cases} x + y = 5 \\ x - y = 1 \end{cases}$$

Como resolver?
- **Soma as duas**: $2x = 6 \Rightarrow x = 3$.
- **Subtrai**: $2y = 4 \Rightarrow y = 2$.

Pronto: $x = 3, y = 2$.

**Pra sistemas grandes**, fica chato fazer no braço. Aí a gente **organiza em matriz**:
$$\begin{pmatrix} 1 & 1 \\ 1 & -1 \end{pmatrix} \begin{pmatrix} x \\ y \end{pmatrix} = \begin{pmatrix} 5 \\ 1 \end{pmatrix}$$

E usa **escalonamento** (modo padrão) ou **regra de Cramer** (com determinantes) ou **inversa** ($\mathbf x = A^{-1}\mathbf b$).

**Quando dá errado?**
- Sistema **sem solução**: as equações se contradizem.
- Sistema com **infinitas soluções**: as equações são redundantes.

</Porta>

<Porta nivel="15" titulo="Pra adolescente de 15 anos">

Sistema linear em forma matricial: empacote os coeficientes em $A$, as incógnitas em $\mathbf{x}$, e os resultados em $\mathbf{b}$. O sistema vira $A\mathbf{x} = \mathbf{b}$.

**Cramer** (didático, pequeno): $x_i = \det(A_i)/\det(A)$. Exemplo:
$$\begin{cases} 2x + y = 5 \\ x - 3y = -2 \end{cases} \Rightarrow A = \begin{pmatrix} 2 & 1 \\ 1 & -3 \end{pmatrix}, \det A = -7$$

$x = \det\begin{pmatrix} 5 & 1 \\ -2 & -3 \end{pmatrix}/(-7) = (-13)/(-7) = 13/7$.

$y = \det\begin{pmatrix} 2 & 5 \\ 1 & -2 \end{pmatrix}/(-7) = (-9)/(-7) = 9/7$.

**Gauss** (padrão): faça operações elementares pra triangularizar. Depois substitui de baixo pra cima.

$$[A | \mathbf b] = \begin{pmatrix} 2 & 1 & | & 5 \\ 1 & 3 & | & 6 \end{pmatrix} \xrightarrow{L_2 \leftarrow L_2 - L_1/2} \begin{pmatrix} 2 & 1 & | & 5 \\ 0 & 5/2 & | & 7/2 \end{pmatrix}$$

Da segunda linha: $y = 7/5$. Substitui na primeira: $2x = 5 - 7/5 \Rightarrow x = 9/5$.

**Classificação**:
- $\det A \neq 0$ → **determinado** (única solução).
- $\det A = 0$ + consistente → **indeterminado** (infinitas).
- $\det A = 0$ + inconsistente → **impossível**.

**Sistema homogêneo** $A\mathbf x = \mathbf 0$: sempre tem $\mathbf x = \mathbf 0$ (trivial). Tem solução não-trivial ⟺ $\det A = 0$.

</Porta>

<Porta nivel="25" titulo="Pra estudante de engenharia">

Resolver $A\mathbf x = \mathbf b$ é a operação **mais comum** de toda computação científica. ~80% do tempo de simulações FEM/CFD é gasto em solvers lineares.

### Hierarquia de solvers

1. **Métodos diretos** (Gauss, LU, QR, Cholesky): $O(n^3)$, exatos a menos de erro de máquina.
2. **Métodos iterativos** (Jacobi, Gauss-Seidel, CG, GMRES, BiCGSTAB): $O(n^2)$ por iter, requer convergência, ideal para esparso.
3. **Métodos especializados**: FFT para circulantes, multigrid para PDEs.

### LU com pivoteamento parcial

$PA = LU$ — $P$ permutação, $L$ triangular inferior unitária, $U$ triangular superior. Resolve $A\mathbf x = \mathbf b$ via:
1. $L\mathbf y = P\mathbf b$ (forward sub, $O(n^2)$).
2. $U\mathbf x = \mathbf y$ (back sub, $O(n^2)$).

Pivoteamento crucial pra estabilidade — evita divisão por números pequenos.

### Sistemas mal-condicionados

$\kappa(A) = \|A\|\|A^{-1}\|$. Para Gauss em float64, perda de $\log_{10}\kappa$ dígitos. Se $\kappa = 10^{16}$, solução é puro lixo.

### Mínimos quadrados (sistema sobredeterminado)

$\min_\mathbf x \|A\mathbf x - \mathbf b\|^2$. Equação normal: $A^TA\mathbf x = A^T\mathbf b$. **Não use!** Condicionamento ao quadrado. Use QR: $A = QR$ ⟹ $R\mathbf x = Q^T\mathbf b$.

### Cross-domínio

- **Circuitos**: KCL/KVL geram sistemas lineares — Modified Nodal Analysis.
- **FEM**: $K\mathbf u = \mathbf f$ — milhões de incógnitas, esparso, simétrico positivo definido.
- **Otimização**: cada iteração de Newton resolve $H\Delta\mathbf x = -\nabla f$.
- **Estatística**: regressão $X^TX\hat\beta = X^T\mathbf y$ (mas use QR/SVD).
- **Filtros (Kalman)**: predição-correção envolve solvers a cada passo.

### Numérico — boas práticas

- **Use `solve`, não `inv`**.
- Para SPD: Cholesky.
- Para esparso: `scipy.sparse.linalg.spsolve` ou iterativos com precondicionador.
- Para dense $> 10^4$: GPU ou métodos iterativos.

</Porta>

Solvers lineares são a **base** de PDEs, ML, otimização e simulação. A escolha do método é decisão arquitetural — pode ser a diferença entre simulação ser viável ou não.

### Classes de matriz e algoritmo ótimo

| Tipo | Solver |
|---|---|
| Densa geral | LU (Gauss + pivot) |
| Densa SPD | Cholesky |
| Densa retangular | QR |
| Esparsa simétrica indef. | MINRES, SYMMLQ |
| Esparsa SPD | CG (com precond.) |
| Esparsa não-simétrica | GMRES, BiCGSTAB |
| Estruturada (Toeplitz) | FFT-based, $O(n \log n)$ |
| Tridiagonal | Thomas, $O(n)$ |
| Banda | LU específica $O(nb^2)$ |

### Iterativos modernos

- **Krylov subspace**: $\mathcal K_k = \text{span}\{\mathbf b, A\mathbf b, \ldots, A^{k-1}\mathbf b\}$. CG (SPD), GMRES (geral) constroem aproximações ótimas em $\mathcal K_k$.
- **Convergência**: $O(\sqrt{\kappa})$ iterações para CG. Precondicionar $\widetilde A = M^{-1}A$ com $\kappa(\widetilde A) \ll \kappa(A)$ é arte.

### Big data / esparso

- **Hypre, PETSc, Trilinos**: bibliotecas industriais para sistemas $> 10^9$.
- **Multigrid**: $O(n)$ para problemas elípticos.
- **Domain decomposition**: paraleliza decomondo $\Omega$ em subdomínios.

### Quant — sistemas em risco

Calibração de modelos (BS, Heston, SABR) gera sistemas mal-condicionados. **Levenberg-Marquardt**, **regularização Tikhonov**, **truncated SVD** são padrão.

### Pesquisa atual

- **Randomized numerical linear algebra**: sketching reduz dim antes de resolver. Drineas, Mahoney.
- **Mixed precision iterative refinement**: solve em FP32, refine em FP64. Ganho 10× em hardware moderno.
- **Quantum algorithms (HHL)**: $O(\log n)$ em qubits para certas classes — ainda teórico.

</Porta>

<Porta nivel="pratica" titulo="Aplicação prática">

**Workflow para resolver $A\mathbf x = \mathbf b$ na prática.**

1. Olhe pra estrutura: simétrica? esparsa? triangular?
2. Escolha o solver compatível.
3. **Nunca use inversa explícita** se o objetivo é resolver.
4. Cheque condicionamento; sempre verifique resíduo $\|A\mathbf x - \mathbf b\|$.

### NumPy receita

```python
import numpy as np
from scipy.linalg import lu, cho_factor, cho_solve, qr, solve_triangular

A = np.array([[2, 1], [1, 3]], dtype=float)
b = np.array([5, 7], dtype=float)

# Padrão
x = np.linalg.solve(A, b)

# Para SPD
c, lo = cho_factor(A)
x = cho_solve((c, lo), b)

# Mínimos quadrados (overdetermined)
A_rect = np.random.randn(100, 5)
b_rect = np.random.randn(100)
x_lsq, residuals, rank, sv = np.linalg.lstsq(A_rect, b_rect, rcond=None)

# Esparso
from scipy.sparse import csr_matrix
from scipy.sparse.linalg import spsolve
A_sp = csr_matrix(A)
x_sp = spsolve(A_sp, b)

# Iterativo (CG para SPD)
from scipy.sparse.linalg import cg
x_cg, info = cg(A_sp, b)
```

### SymPy simbólico

```python
import sympy as sp

```

### Cramer manual 2×2

```python
def cramer_2x2(A, b):
    detA = A[0,0]*A[1,1] - A[0,1]*A[1,0]
    if abs(detA) < 1e-14:
        return None
    Ax = np.array([[b[0], A[0,1]], [b[1], A[1,1]]])
    Ay = np.array([[A[0,0], b[0]], [A[1,0], b[1]]])
    return np.array([np.linalg.det(Ax)/detA, np.linalg.det(Ay)/detA])
```

### Pegadinhas

- Se `solve` der erro `LinAlgError: singular matrix`: use `lstsq` ou `pinv`.
- Solução pode existir mesmo com $\det \approx 0$ — depende de $\mathbf b$ estar no espaço-coluna.
- **Sempre verifique** resíduo: `np.linalg.norm(A @ x - b)`.
