<Porta nivel="formal" titulo="Definições">

### Fatorial

$n! = n \cdot (n-1) \cdot (n-2) \cdots 1$. Convenção: $0! = 1$.

Crescimento:

| $n$ | $n!$ |
|---|---|
| 5 | 120 |
| 10 | 3 628 800 |
| 20 | $\approx 2{,}4 \times 10^{18}$ |
| 70 | $> 10^{100}$ |
| 170 | overflow em float64 |

Aproximação de Stirling: $n! \approx \sqrt{2\pi n}(n/e)^n$.

### Permutação simples

$P_n = n!$ — formas de ordenar $n$ objetos distintos em fila.

### Permutação com repetição

Para $n$ objetos com $n_1$ do tipo 1, $n_2$ do tipo 2, ..., $n_k$ do tipo $k$:
$$P_n^{n_1, n_2, \ldots, n_k} = \frac{n!}{n_1! \cdot n_2! \cdots n_k!}$$

Anagramas de "ARARA" (3 A's, 2 R's): $5!/(3! \cdot 2!) = 10$.

### Arranjo simples

$$A_n^p = n(n-1)(n-2) \cdots (n-p+1) = \frac{n!}{(n-p)!}$$

Formas de ordenar $p$ objetos selecionados de $n$ disponíveis.

### Permutação circular

$n$ objetos em círculo: $(n-1)!$. Razão: a "primeira posição" é arbitrária.

### Diferença entre permutação e arranjo

- **Permutação**: usa todos os $n$ objetos.
- **Arranjo**: seleciona $p \leq n$ e os ordena.

Quando $p = n$: arranjo coincide com permutação.

</Porta>

<Porta nivel="5" titulo="Pra criança de 5 anos">

Imagina que você tem **3 brinquedos** numa fileira. De quantos jeitos você pode arrumar eles?

Pode pôr A, B, C. Ou A, C, B. Ou B, A, C. Ou B, C, A. Ou C, A, B. Ou C, B, A. **6 jeitos**!

Esse "6" é $3 \times 2 \times 1$ — fatorial de 3, escrito $3!$. Pra cada novo brinquedo, multiplica.

</Porta>

<Porta nivel="10" titulo="Pra criança de 10 anos">

**Permutação** = mexer na ordem de um grupo todo.

Exemplo: você tem 5 livros e quer arrumar na estante. **Quantas formas?** $5! = 5 \times 4 \times 3 \times 2 \times 1 = 120$.

**Arranjo** é parecido, mas você só usa **alguns** dos itens, não todos.

Exemplo: você tem 10 atletas e vai dar pódio (1.º, 2.º, 3.º). **$10 \times 9 \times 8 = 720$**.

**Atenção**: nas duas, a **ordem importa**. Trocar A com B no pódio é diferente.

**Quando letras se repetem**: anagramas de PARA = $4!/2! = 12$ (porque a letra A repete 2 vezes — divide).

**Mesa redonda** é especial: 5 pessoas em volta da mesa = $4! = 24$ (não $5!$). Por quê? Girar todos juntos não conta como nova arrumação.

</Porta>

<Porta nivel="15" titulo="Pra adolescente de 15 anos">

**Permutação** = ordenar tudo. Quantas formas de organizar 5 livros em uma estante? $5! = 120$.

**Arranjo** = ordenar uma seleção. Quantas formas de premiar com 1.º, 2.º e 3.º lugar entre 10 atletas? $A_{10}^3 = 10 \cdot 9 \cdot 8 = 720$.

**Pra usar a fórmula direto**: nas duas a ORDEM IMPORTA. Trocar A com B em primeiro e segundo é diferente.

**Permutação com repetição** é especial: anagramas de "AMOR" são $4! = 24$, mas anagramas de "AMA" são $3!/2! = 3$ (não $6$, porque AA é igual a AA).

**Casos clássicos**:
- "MISSISSIPPI" (11 letras: 1M, 4I, 4S, 2P): $11!/(1!\cdot 4!\cdot 4!\cdot 2!) = 34\,650$.
- Mesa redonda com 8 pessoas: $7! = 5\,040$.

**Fatorial cresce rápido**: $5! = 120$, $10! \approx 3{,}6$ milhões, $20!$ já é maior que $10^{18}$. **Por isso problemas envolvendo permutação são "explosivos"**.

**Arranjo vs PFC**: $A_n^p = n(n-1)(n-2)\cdots(n-p+1)$ é só PFC com restrição "sem repetir".

</Porta>

<Porta nivel="25" titulo="Pra estudante de engenharia">

Permutações e arranjos são primitivas combinatórias para enumeração e estimação probabilística.

### Demonstração: $P_n = n!$

PFC: 1ª posição $n$ opções, 2ª posição $n-1$, ..., última 1. Produto: $n!$.

### Demonstração: arranjo via PFC

$A_n^p = n(n-1)(n-2)\cdots(n-p+1) = n!/(n-p)!$.

### Crescimento de fatorial

Stirling: $\ln(n!) = n\ln n - n + O(\ln n)$. Daí $n! \sim \sqrt{2\pi n}(n/e)^n$. Em ML, importante pra log-likelihood Gaussiana, entropia de modelos categóricos.

### Cross-domínio

- **TSP (caixeiro viajante)**: $n!$ rotas → NP-hard. Heurísticas (2-opt, Lin-Kernighan, Concorde) atacam casos práticos.
- **Embaralhamento de baralho**: $52! \approx 8\times 10^{67}$ — Riffle shuffle leva $\log_2(52!) \approx 226$ riffles para uniformizar (Bayer-Diaconis: 7 riffles bastam pra "praticamente" uniforme).
- **Ordenamento**: pior caso de comparações para sort = $\Omega(n \log n)$ porque árvore de decisão tem $\geq n!$ folhas.
- **Bioinformática**: alinhamento de sequências envolve permutações; criptografia de cifras-substituição é permutação de alfabeto.

### Permutação como elemento de $S_n$

$S_n$ é grupo simétrico de ordem $n!$. Estrutura grupal — composição, ciclos, sinal — é central em álgebra abstrata, teoria de Galois, física.

- **Sinal**: $\text{sgn}(\sigma) \in \{+1, -1\}$ via paridade do número de transposições.
- **Decomposição em ciclos**: única a menos de ordem.
- **Aparece em $\det$**: $\det A = \sum_\sigma \text{sgn}(\sigma) \prod a_{i,\sigma(i)}$.

### Aplicações em ML/quant

- **Permutation tests**: significância estatística sem assumir normalidade.
- **Random forests**: subsampling de features e dados via permutações.
- **Cross-validation**: shuffle splits.
- **MCMC**: muitas cadeias percorrem $S_n$ (TSP por simulated annealing).

<Porta nivel="40" titulo="Pra profissional sênior">

Permutações são instâncias de **grupo simétrico** $S_n$ — objeto central em álgebra, geometria e física.

### Wolfram tour

```mathematica
Permutations[Range[5]]
PermutationCycles[Cycles[{{1, 2, 3}, {4, 5}}]]
SignaturePermutation[{2, 3, 1, 5, 4}]
GroupOrder[SymmetricGroup[5]]
```

### Estrutura do $S_n$

- $|S_n| = n!$.
- **Geradores**: transposições adjacentes $(i, i+1)$.
- $S_n$ não-abeliano para $n \geq 3$.
- **Subgrupo alternado** $A_n$: permutações de sinal +1, $|A_n| = n!/2$.
- $S_n$ não-solúvel para $n \geq 5$ — daí impossibilidade de fórmula geral pra raízes de polinômios de grau ≥ 5 (Galois).

### Quant — apreçamento

- **Modelos de Markov ocultos**: estados ocultos formam alfabeto, observações dependem. Permutação de rótulos é simetria do modelo.
- **Stochastic discount factor**: enumeração de cenários históricos por permutação para bootstrap.

### Permutações em ML moderno

- **Transformer attention** é equivariante por permutação dos tokens — daí position encoding.
- **Permutation-invariant networks**: DeepSets, PointNet — agregam features via soma/max.
- **Sinkhorn-Knopp**: aprende permutação suave via doubly stochastic matrices.

### Combinatorial generation

- **Lehmer code**: bijeção $S_n \leftrightarrow \prod \{0, \ldots, n-1\}$. Permite indexação aleatória sem gerar lista.
- **Heap's algorithm**: gera todas as $n!$ permutações em $O(n!)$ tempo total.
- **Steinhaus-Johnson-Trotter**: permutações em ordem que muda 2 elementos adjacentes por vez (Gray code para $S_n$).

### Pesquisa

- **Pattern avoidance** em permutações — Wilf-equivalence, Stanley-Wilf conjecture.
- **Aleatorização**: misturas de baralhos, cutoff phenomenon (Diaconis).
- **Permutation polytopes** — geometria.

</Porta>

<Porta nivel="pratica" titulo="Aplicação prática">

**Workflow para permutação/arranjo.**

1. **Ordem importa?** Se sim, aqui é o lugar.
2. **Usa todos os elementos?** Permutação. **Subconjunto?** Arranjo.
3. **Tem repetição?** Divida pelos fatoriais das multiplicidades.
4. **Configuração circular?** $(n-1)!$.

### Python receita

```python
import math
import itertools

# Fatorial
math.factorial(10)        # 3628800

# Arranjo A(n, p)
math.perm(10, 3)          # 720

# Permutações geradas
list(itertools.permutations([1, 2, 3]))   # 6 tuplas

# Permutações de tamanho k
list(itertools.permutations([1, 2, 3, 4], 2))   # A(4,2) = 12

# Permutação com repetição: gere e tire duplicatas
def perm_with_rep(letters):
    return set(itertools.permutations(letters))

print(len(perm_with_rep("ARARA")))    # 10

# Stirling para fatorial grande
def stirling(n):
    import math
    return math.sqrt(2*math.pi*n) * (n/math.e)**n
```

### SymPy

```python
import sympy as sp
sp.factorial(20)              # exato, 2.4e18
sp.binomial(10, 3) * sp.factorial(3)   # arranjo via combinação × p!
```

### Mesa redonda e similares

```python
def round_table(n):
    return math.factorial(n - 1)

print(round_table(8))   # 5040
```

### Pegadinhas

- **`A == B` em string** vs **`A == B` em multiset**: anagramas com letras iguais precisam de tratamento especial.
- **Indexação 0 vs 1** em permutações.
- **Overflow**: $n! > 10^{300}$ para $n > 170$. Use `math.factorial` (Python int arbitrário) ou logaritmos.
- **"Pelo menos um juntos"** geralmente é mais fácil pelo complementar.
