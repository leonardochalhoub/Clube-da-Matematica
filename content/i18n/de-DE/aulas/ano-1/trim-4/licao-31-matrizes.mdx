<Porta nivel="formal" titulo="Definition und Typen">

<Definicao titulo="Matrix">
Eine $m \times n$-Matrix über $\mathbb{R}$ ist eine Funktion $A: \{1, \ldots, m\} \times \{1, \ldots, n\} \to \mathbb{R}$. Notation $A = (a_{ij})$ oder $A_{m \times n}$.

Gleichheit: $A = B \iff$ gleiche Dimension und $a_{ij} = b_{ij}$ für alle $i, j$.
</Definicao>

### Spezielle Typen

| Typ | Definition |
|---|---|
| Quadratisch | $m = n$ |
| Zeile | $1 \times n$ |
| Spalte | $m \times 1$ |
| Diagonal | quadratisch mit $a_{ij} = 0$ für $i \neq j$ |
| Einheitsmatrix $I_n$ | Diagonal mit $a_{ii} = 1$ |
| Nullmatrix $O$ | alle Elemente null |
| Obere Dreiecks- | $a_{ij} = 0$ für $i > j$ |
| Untere Dreiecks- | $a_{ij} = 0$ für $i < j$ |
| Symmetrisch | $A^T = A$, also $a_{ij} = a_{ji}$ |
| Antisymmetrisch | $A^T = -A$, also $a_{ij} = -a_{ji}$ |
| Skalar | Diagonal mit $a_{ii} = k$ konstant |

### Diagonale einer quadratischen Matrix

Die **Hauptdiagonale** ist $\{a_{ii}\}$. **Spur**: $\text{tr}(A) = \sum_{i=1}^n a_{ii}$.

### Bildungsgesetz

Häufig definiert man $A$ über die Formel $a_{ij} = f(i, j)$. Beispiele:
- $a_{ij} = i + j$
- $a_{ij} = (-1)^{i+j}$
- $a_{ij} = \delta_{ij}$ (Kronecker-Delta — erzeugt die Einheitsmatrix)

</Porta>

<Porta nivel="5" titulo="Für ein 5-jähriges Kind">

Stell dir einen **Eierkarton** vor. Er hat Reihen und Spalten. Jeder Platz hält ein Ei (das ist eine Zahl).

Eine **Matrix** ist genau das: eine quadratische oder rechteckige Box voller Zahlen, ordentlich angeordnet — jede an ihrem Platz. Um zu sagen, wo eine Zahl ist, nennst du Reihe und Spalte: „Zeile 2, Spalte 3" — gefunden!

</Porta>

<Porta nivel="10" titulo="Für ein 10-jähriges Kind">

Du hast schon **Tabellen** in der Schule gesehen. Eine Notentabelle: jeder Schüler eine Zeile, jedes Fach eine Spalte. **Matrix ist genau das** — eine Tabelle mit Zahlen.

Beispiele aus dem Alltag:
- **Zeugnis**: 30 Schüler × 5 Fächer = Matrix 30×5.
- **Computerbildschirm**: Jedes Pixel hat einen Farbwert. Ein Bildschirm 1920×1080 ist eine riesige Matrix.
- **Entfernungstabelle**: Stadt × Stadt. Zeile München, Spalte Berlin = 585 km.

Jede Zahl hat eine **Adresse**: $a_{ij}$ heißt „Zeile $i$, Spalte $j$". Das $i$ kommt immer zuerst.

Spezielle Typen, die häufig vorkommen:
- **Quadratisch**: gleich viele Zeilen wie Spalten.
- **Einheitsmatrix**: wie die Zahl 1 — Einsen auf der Diagonale, sonst Nullen.
- **Nullmatrix**: nur Nullen.

</Porta>

<Porta nivel="15" titulo="Für einen 15-jährigen Jugendlichen">

Eine Matrix ist eine **Tabelle** von Zahlen, organisiert in Zeilen und Spalten:

$$A = \begin{pmatrix} 1 & 2 & 3 \\ 4 & 5 & 6 \end{pmatrix}$$

Das ist eine Matrix $2 \times 3$ (2 Zeilen, 3 Spalten). Jede Zahl hat eine **Adresse**: $a_{12} = 2$ (Zeile 1, Spalte 2). $a_{23} = 6$ (Zeile 2, Spalte 3).

**Wofür?** Um Daten zu speichern und zu organisieren, vor allem um **Massenberechnungen** durchzuführen:
- Schülerergebnisse × Fächer: Schüler-Fach-Matrix.
- Entfernungen zwischen Städten: symmetrische Matrix.
- Pixel eines Bildes: Intensitätsmatrix.
- Koeffizienten eines linearen Systems: Koeffizientenmatrix.
- Gewichte eines neuronalen Netzes: Vektoren und Matrizen (tonnenweise).

**Gleichheit**: Zwei Matrizen sind gleich, wenn sie **dieselbe Dimension** UND **alle entsprechenden Elemente gleich** haben. Man kann nicht $2 \times 3$ mit $3 \times 2$ vergleichen.

**Wichtige Typen**:
- **Quadratisch**: gleiche Anzahl Zeilen und Spalten.
- **Einheitsmatrix** $I_n$: 1en auf der Diagonale, sonst 0. Wirkt wie die „1" der Multiplikation ($IA = A$).
- **Diagonal**: nur Nullen außerhalb der Hauptdiagonale.
- **Symmetrisch**: $A^T = A$ — perfekter Spiegel an der Diagonale.

**Bildungsgesetz**: Man kann eine Matrix aus einer Formel aufbauen. Wenn $a_{ij} = i + j$, dann $a_{12} = 3$, $a_{21} = 3$, $a_{33} = 6$ usw.

</Porta>

<Porta nivel="25" titulo="Für einen Ingenieurstudenten">

Die Matrix ist das **Grundobjekt** des wissenschaftlichen Rechnens und des ML. In Python/NumPy ist „alles ein Array". In MATLAB ist „alles eine Matrix". Der Begriff stammt von **Cayley (1858)**, die Struktur war aber bereits bei Lagrange und Gauß implizit vorhanden.

### Sicht als lineare Abbildung

Eine Matrix $A_{m \times n}$ stellt eine lineare Abbildung $T: \mathbb{R}^n \to \mathbb{R}^m$ via $T(\mathbf x) = A\mathbf x$ dar. Die **Spalten** sind Bilder der Vektoren der kanonischen Basis. Diese Interpretation macht Matrizen allgegenwärtig.

### Häufige Matrixstrukturen

- **Dünn besetzt**: Großteil der Elemente ist null. Formate CSR/CSC speichern nur die Nichtnull-Einträge. Entscheidend in FEM, Netzwerken, NLP, Graphen mit Millionen Knoten.
- **Bandmatrix**: Nullen außerhalb eines zentralen Bandes. Nützlich für Tridiagonalsysteme (Wärme, Crank-Nicolson, kubische Splines).
- **Dicht**: alles gespeichert. Standard in ML, reiner linearer Algebra, BLAS-3.
- **Block-Diagonal**: quadratische Blöcke auf der Diagonale — entkoppelte Systeme.
- **Toeplitz**: $a_{ij}$ hängt nur von $i - j$ ab — Faltungen, Signalverarbeitung.

### Anwendungen über Domänen hinweg

- **Regelungstechnik**: Zustand $\dot{\mathbf{x}} = A\mathbf{x} + B\mathbf{u}$, $\mathbf y = C\mathbf x$ — A ist die „Dynamik".
- **Strukturen (FEM)**: globale Steifigkeitsmatrix $K$ ist dünn besetzt, symmetrisch und positiv definit.
- **Digitalbild**: jeder RGB-Kanal ist eine $H \times W$-Matrix mit Intensitäten $\{0, \ldots, 255\}$.
- **Neuronale Netze**: jede dichte Schicht ist $\mathbf y = \sigma(W\mathbf x + \mathbf b)$.
- **Quanten**: Zustände sind Vektoren, Observablen sind hermitesche Matrizen $A = A^*$.
- **Statistik**: Kovarianzmatrix $\Sigma$ ist symmetrisch positiv semidefinit.
- **Graphen**: Adjazenzmatrix $A_{ij} = 1$, falls eine Kante existiert — Eigenwerte kodieren die Struktur.

### Vektorraum $M_{m \times n}(\mathbb R)$

Die Menge der $m \times n$-Matrizen ist ein Vektorraum der **Dimension** $mn$ — kanonische Basis $\{E_{ij}\}$, wobei $E_{ij}$ eine 1 an Position $(i,j)$ und sonst 0 hat.

</Porta>

<Porta nivel="40" titulo="Für einen erfahrenen Profi">

Matrizen sind endliche Instanzen von **linearen Operatoren** in Vektorräumen. Die allgemeine Theorie lebt in $\text{Hom}(V, W)$ für Vektorräume $V, W$ — bei Wahl von Basen $\text{Hom}(V, W) \cong M_{m \times n}$.

### Strukturelle Sicht

- **$M_n(\mathbb R)$ ist eine assoziative, nicht kommutative Algebra** mit Einheit $I_n$.
- **Spektralzerlegungen**: SVD ($A = U\Sigma V^T$), Eigenwerte ($A = PDP^{-1}$, falls diagonalisierbar), Schur, Jordan — jede offenbart einen anderen Aspekt.
- **Matrixgruppen** ($GL_n$, $SO_n$, $U_n$, $Sp_{2n}$) sind zentrale Objekte in Geometrie, Physik und Darstellungstheorie.

### Modernes Numerisches

- **BLAS/LAPACK**: GEMM (Multiplikation) ist der am stärksten optimierte Kernel des Rechnens. Theoretisch $O(n^3)$, doch Speicher/Cache dominieren — Flops sind auf der GPU „gratis".
- **Autodiff und ML**: Bibliotheken (PyTorch, JAX) behandeln Tensoren als verallgemeinerte Matrizen. Backpropagation ist die Kettenregel auf Graphen matrixwertiger Operationen.
- **Randomisierung**: zufällige Projektionen (Johnson-Lindenstrauss), Sketching, randomisiertes SVD reduzieren die Komplexität bei Big Data.

### Quant/Finance

- **Kovarianzmatrix** $\Sigma$ der Renditen ist Grundlage für Markowitz. Robuste Schätzung (Shrinkage, Ledoit-Wolf) ist ein zentrales Problem.
- **Übergangsmatrizen** in Markov-Ketten modellieren Kreditratings (S&P, Moody's).
- **Cholesky** $\Sigma = LL^T$ erzeugt korrelierte Gauß-Stichproben in Monte Carlo.

### Verallgemeinerungen

- **Tensoren** ($T_{ijk\ldots}$): verallgemeinern Matrizen auf mehrere Indizes. Zerlegungen (CP, Tucker) sind Analoga zur SVD.
- **Operatoren in unendlicher Dimension**: Fredholm-Integrale, kompakte Operatoren — kontinuierliche Analoga.
- **Matrizen über endlichen Körpern**: fehlerkorrigierende Codes, Kryptographie (Niederreiter, McEliece), Kombinatorik.

</Porta>

<Porta nivel="pratica" titulo="Praktische Anwendung">

**Workflow für die praktische Arbeit mit Matrizen.**

1. **Struktur identifizieren**: Dimension? symmetrisch? dünn besetzt? Dreiecks-? Struktur spart Speicher und Flops.
2. **Repräsentation wählen**: NumPy-Array (dicht), `scipy.sparse` (dünn), `pandas.DataFrame` (mit Labels), `xarray` (mehrdim mit Namen).
3. **Vektorisieren**: niemals `for`-Schleifen für elementweise Operationen — Broadcasting verwenden.

### NumPy — Rezept

```python
import numpy as np

# Konstruktion
A = np.array([[1, 2, 3], [4, 5, 6]])     # 2x3 explizit
I = np.eye(3)                             # Einheitsmatrix 3x3
Z = np.zeros((4, 5))                      # Nullmatrix 4x5
D = np.diag([2, -1, 5])                   # Diagonal

# Bildungsgesetz
i, j = np.indices((3, 3))
A_gesetz = i + j                          # a_ij = i + j

# Zugriff
print(A[0, 2])      # Element Zeile 0 Spalte 2 (0-basierte Indizierung!)
print(A.shape)      # (2, 3)
print(A.T)          # Transponierte
print(np.trace(np.eye(5)))   # Spur
```

### SymPy — symbolisch

```python
import sympy as sp
A = sp.Matrix([[1, 2], [3, 4]])
print(A.shape, A.T, A.trace())
```

### Symmetrie prüfen

```python
def is_symmetric(A, tol=1e-9):
    return np.allclose(A, A.T, atol=tol)
```

### Häufige Fallstricke

- **0-basierte Indizierung** in Python/NumPy vs. **1-basiert** in der Mathematik. Vorsicht beim Übersetzen von Formeln.
- **Zeile vs. Spalte**: `A[i]` ist Zeile $i$. Für eine Spalte `A[:, j]`.
- **Kopie vs. View**: `B = A` kopiert nicht! Verwenden Sie `B = A.copy()`.
- **`np.matrix` ist veraltet**. Verwenden Sie `np.ndarray` (`@` für Matrixmultiplikation).
