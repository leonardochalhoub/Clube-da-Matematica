<Porta nivel="formal" titulo="Métodos de resolución">

### Forma matricial

$\begin{cases} a_1 x + b_1 y + c_1 z = d_1 \\ a_2 x + b_2 y + c_2 z = d_2 \\ a_3 x + b_3 y + c_3 z = d_3 \end{cases}$ ⟺ $A\mathbf{x} = \mathbf{b}$ con $A_{3 \times 3}$, $\mathbf{x}, \mathbf{b} \in \mathbb{R}^3$.

### Método 1 — Eliminación de Gauss

Operaciones elementales (no alteran la solución):
1. Intercambiar dos filas.
2. Multiplicar una fila por escalar no nulo.
3. Sumar un múltiplo de una fila a otra.

Objetivo: triangularizar $[A | \mathbf{b}]$ hasta forma escalonada. Después, sustitución regresiva.

### Método 2 — Cramer

Para $A\mathbf{x} = \mathbf{b}$ con $\det A \neq 0$:
$$x_i = \frac{\det A_i}{\det A}$$

donde $A_i$ es $A$ con la $i$-ésima columna sustituida por $\mathbf{b}$.

### Método 3 — Inversa

$\mathbf{x} = A^{-1} \mathbf{b}$. Se puede calcular $A^{-1}$ vía $[A | I] \to [I | A^{-1}]$ por eliminación.

### Cuándo usar cada uno

| Método | Coste | Cuándo usar |
|---|---|---|
| Cramer | $O(n^4)$ | $n \leq 3$, didáctico |
| Gauss | $O(n^3)$ | estándar práctico |
| Inversa | $O(n^3)$ + $O(n^2)$/sistema | múltiples $\mathbf b$ con el mismo $A$ |
| LU | $O(n^3)$ + $O(n^2)$/sistema | múltiples sistemas, mejor que la inversa |

### Clasificación de sistemas

- **Determinado (SCD)**: solución única ($\det A \neq 0$, o $\text{rank}(A) = \text{rank}([A|\mathbf b]) = n$).
- **Indeterminado (SCI)**: infinitas soluciones ($\text{rank}(A) = \text{rank}([A|\mathbf b]) < n$).
- **Imposible (SI)**: sin solución ($\text{rank}(A) < \text{rank}([A|\mathbf b])$).

### Teorema de Rouché-Capelli (Rouché-Frobenius)

$A\mathbf x = \mathbf b$ tiene solución ⟺ $\text{rank}(A) = \text{rank}([A|\mathbf b])$. La solución es única si ambos = número de incógnitas.

</Porta>

<Porta nivel="5" titulo="Para un niño de 5 años">

Un sistema es **un montón de ecuaciones a la vez**. Tipo: "tengo 3 manzanas y 2 peras, en total 13 €. Tengo 1 manzana y 4 peras, en total 16 €. ¿Cuánto cuesta cada una?"

Para resolverlo, **juntas** las ecuaciones de un modo astuto para **eliminar** una de las frutas, descubres el precio de la otra y vuelves. Es un rompecabezas con receta segura.

</Porta>

<Porta nivel="10" titulo="Para un niño de 10 años">

**Sistema lineal** = varias ecuaciones con varias incógnitas que tienen que ser ciertas a la vez.

Ejemplo:
$$\begin{cases} x + y = 5 \\ x - y = 1 \end{cases}$$

¿Cómo resolver?
- **Suma las dos**: $2x = 6 \Rightarrow x = 3$.
- **Resta**: $2y = 4 \Rightarrow y = 2$.

Listo: $x = 3, y = 2$.

**Para sistemas grandes**, es pesado hacerlo a mano. Entonces lo **organizamos en una matriz**:
$$\begin{pmatrix} 1 & 1 \\ 1 & -1 \end{pmatrix} \begin{pmatrix} x \\ y \end{pmatrix} = \begin{pmatrix} 5 \\ 1 \end{pmatrix}$$

Y se usa **eliminación** (modo estándar), **regla de Cramer** (con determinantes) o **inversa** ($\mathbf x = A^{-1}\mathbf b$).

**¿Cuándo sale mal?**
- Sistema **sin solución**: las ecuaciones se contradicen.
- Sistema con **infinitas soluciones**: las ecuaciones son redundantes.

</Porta>

<Porta nivel="15" titulo="Para un adolescente de 15 años">

Sistema lineal en forma matricial: empaqueta los coeficientes en $A$, las incógnitas en $\mathbf{x}$, y los resultados en $\mathbf{b}$. El sistema se vuelve $A\mathbf{x} = \mathbf{b}$.

**Cramer** (didáctico, pequeño): $x_i = \det(A_i)/\det(A)$. Ejemplo:
$$\begin{cases} 2x + y = 5 \\ x - 3y = -2 \end{cases} \Rightarrow A = \begin{pmatrix} 2 & 1 \\ 1 & -3 \end{pmatrix}, \det A = -7$$

$x = \det\begin{pmatrix} 5 & 1 \\ -2 & -3 \end{pmatrix}/(-7) = (-13)/(-7) = 13/7$.

$y = \det\begin{pmatrix} 2 & 5 \\ 1 & -2 \end{pmatrix}/(-7) = (-9)/(-7) = 9/7$.

**Gauss** (estándar): aplica operaciones elementales para triangularizar. Después sustituye de abajo hacia arriba.

$$[A | \mathbf b] = \begin{pmatrix} 2 & 1 & | & 5 \\ 1 & 3 & | & 6 \end{pmatrix} \xrightarrow{L_2 \leftarrow L_2 - L_1/2} \begin{pmatrix} 2 & 1 & | & 5 \\ 0 & 5/2 & | & 7/2 \end{pmatrix}$$

De la segunda fila: $y = 7/5$. Sustituye en la primera: $2x = 5 - 7/5 \Rightarrow x = 9/5$.

**Clasificación**:
- $\det A \neq 0$ → **determinado** (única solución).
- $\det A = 0$ + consistente → **indeterminado** (infinitas).
- $\det A = 0$ + inconsistente → **imposible**.

**Sistema homogéneo** $A\mathbf x = \mathbf 0$: siempre tiene $\mathbf x = \mathbf 0$ (trivial). Tiene solución no trivial ⟺ $\det A = 0$.

</Porta>

<Porta nivel="25" titulo="Para un estudiante de ingeniería">

Resolver $A\mathbf x = \mathbf b$ es la operación **más común** de toda la computación científica. ~80% del tiempo de simulaciones FEM/CFD se gasta en solvers lineales.

### Jerarquía de solvers

1. **Métodos directos** (Gauss, LU, QR, Cholesky): $O(n^3)$, exactos salvo error de máquina.
2. **Métodos iterativos** (Jacobi, Gauss-Seidel, CG, GMRES, BiCGSTAB): $O(n^2)$ por iter, requiere convergencia, ideal para disperso.
3. **Métodos especializados**: FFT para circulantes, multigrid para PDEs.

### LU con pivoteo parcial

$PA = LU$: $P$ permutación, $L$ triangular inferior unitaria, $U$ triangular superior. Resuelve $A\mathbf x = \mathbf b$ vía:
1. $L\mathbf y = P\mathbf b$ (forward sub, $O(n^2)$).
2. $U\mathbf x = \mathbf y$ (back sub, $O(n^2)$).

Pivoteo crucial para estabilidad: evita la división por números pequeños.

### Sistemas mal condicionados

$\kappa(A) = \|A\|\|A^{-1}\|$. Para Gauss en float64, pérdida de $\log_{10}\kappa$ dígitos. Si $\kappa = 10^{16}$, la solución es pura basura.

### Mínimos cuadrados (sistema sobredeterminado)

$\min_\mathbf x \|A\mathbf x - \mathbf b\|^2$. Ecuación normal: $A^TA\mathbf x = A^T\mathbf b$. **¡No la uses!** Condicionamiento al cuadrado. Usa QR: $A = QR$ ⟹ $R\mathbf x = Q^T\mathbf b$.

### Cross-dominio

- **Circuitos**: KCL/KVL generan sistemas lineales: Modified Nodal Analysis.
- **FEM**: $K\mathbf u = \mathbf f$: millones de incógnitas, disperso, simétrico positivo definido.
- **Optimización**: cada iteración de Newton resuelve $H\Delta\mathbf x = -\nabla f$.
- **Estadística**: regresión $X^TX\hat\beta = X^T\mathbf y$ (pero usa QR/SVD).
- **Filtros (Kalman)**: predicción-corrección implica solvers en cada paso.

### Numérico — buenas prácticas

- **Usa `solve`, no `inv`**.
- Para SPD: Cholesky.
- Para disperso: `scipy.sparse.linalg.spsolve` o iterativos con precondicionador.
- Para denso $> 10^4$: GPU o métodos iterativos.

</Porta>

<Porta nivel="40" titulo="Para un profesional sénior">

Los solvers lineales son la **base** de PDEs, ML, optimización y simulación. La elección del método es decisión arquitectural: puede ser la diferencia entre que la simulación sea viable o no.

### Clases de matriz y algoritmo óptimo

| Tipo | Solver |
|---|---|
| Densa general | LU (Gauss + pivot) |
| Densa SPD | Cholesky |
| Densa rectangular | QR |
| Dispersa simétrica indef. | MINRES, SYMMLQ |
| Dispersa SPD | CG (con precond.) |
| Dispersa no simétrica | GMRES, BiCGSTAB |
| Estructurada (Toeplitz) | FFT-based, $O(n \log n)$ |
| Tridiagonal | Thomas, $O(n)$ |
| Banda | LU específica $O(nb^2)$ |

### Iterativos modernos

- **Krylov subspace**: $\mathcal K_k = \text{span}\{\mathbf b, A\mathbf b, \ldots, A^{k-1}\mathbf b\}$. CG (SPD), GMRES (general) construyen aproximaciones óptimas en $\mathcal K_k$.
- **Convergencia**: $O(\sqrt{\kappa})$ iteraciones para CG. Precondicionar $\widetilde A = M^{-1}A$ con $\kappa(\widetilde A) \ll \kappa(A)$ es arte.

### Big data / disperso

- **Hypre, PETSc, Trilinos**: bibliotecas industriales para sistemas $> 10^9$.
- **Multigrid**: $O(n)$ para problemas elípticos.
- **Domain decomposition**: paraleliza descomponiendo $\Omega$ en subdominios.

### Quant — sistemas en riesgo

La calibración de modelos (BS, Heston, SABR) genera sistemas mal condicionados. **Levenberg-Marquardt**, **regularización Tikhonov**, **truncated SVD** son estándar.

### Investigación actual

- **Randomized numerical linear algebra**: sketching reduce dimensión antes de resolver. Drineas, Mahoney.
- **Mixed precision iterative refinement**: solve en FP32, refine en FP64. Ganancia 10× en hardware moderno.
- **Quantum algorithms (HHL)**: $O(\log n)$ en qubits para ciertas clases: aún teórico.

</Porta>

<Porta nivel="pratica" titulo="Aplicación práctica">

**Workflow para resolver $A\mathbf x = \mathbf b$ en la práctica.**

1. Mira la estructura: ¿simétrica? ¿dispersa? ¿triangular?
2. Elige el solver compatible.
3. **Nunca uses inversa explícita** si el objetivo es resolver.
4. Comprueba el condicionamiento; verifica siempre el residuo $\|A\mathbf x - \mathbf b\|$.

### NumPy receta

```python
import numpy as np
from scipy.linalg import lu, cho_factor, cho_solve, qr, solve_triangular

A = np.array([[2, 1], [1, 3]], dtype=float)
b = np.array([5, 7], dtype=float)

# Estándar
x = np.linalg.solve(A, b)

# Para SPD
c, lo = cho_factor(A)
x = cho_solve((c, lo), b)

# Mínimos cuadrados (overdetermined)
A_rect = np.random.randn(100, 5)
b_rect = np.random.randn(100)
x_lsq, residuals, rank, sv = np.linalg.lstsq(A_rect, b_rect, rcond=None)

# Disperso
from scipy.sparse import csr_matrix
from scipy.sparse.linalg import spsolve
A_sp = csr_matrix(A)
x_sp = spsolve(A_sp, b)

# Iterativo (CG para SPD)
from scipy.sparse.linalg import cg
x_cg, info = cg(A_sp, b)
```

### SymPy simbólico

```python
import sympy as sp

```

### Cramer manual 2×2

```python
def cramer_2x2(A, b):
    detA = A[0,0]*A[1,1] - A[0,1]*A[1,0]
    if abs(detA) < 1e-14:
        return None
    Ax = np.array([[b[0], A[0,1]], [b[1], A[1,1]]])
    Ay = np.array([[A[0,0], b[0]], [A[1,0], b[1]]])
    return np.array([np.linalg.det(Ax)/detA, np.linalg.det(Ay)/detA])
```

### Trampas

- Si `solve` da error `LinAlgError: singular matrix`: usa `lstsq` o `pinv`.
- La solución puede existir incluso con $\det \approx 0$: depende de que $\mathbf b$ esté en el espacio columna.
- **Verifica siempre** el residuo: `np.linalg.norm(A @ x - b)`.
