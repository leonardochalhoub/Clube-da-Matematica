<Porta nivel="formal" titulo="Definición y tipos">

<Definicao titulo="Matriz">
Matriz $m \times n$ sobre $\mathbb{R}$ es una función $A: \{1, \ldots, m\} \times \{1, \ldots, n\} \to \mathbb{R}$. Notación $A = (a_{ij})$ o $A_{m \times n}$.

Igualdad: $A = B \iff$ misma dimensión y $a_{ij} = b_{ij}$ para todos $i, j$.
</Definicao>

### Tipos especiales

| Tipo | Definición |
|---|---|
| Cuadrada | $m = n$ |
| Fila | $1 \times n$ |
| Columna | $m \times 1$ |
| Diagonal | cuadrada con $a_{ij} = 0$ para $i \neq j$ |
| Identidad $I_n$ | diagonal con $a_{ii} = 1$ |
| Nula $O$ | todos los elementos nulos |
| Triangular superior | $a_{ij} = 0$ para $i > j$ |
| Triangular inferior | $a_{ij} = 0$ para $i < j$ |
| Simétrica | $A^T = A$, es decir $a_{ij} = a_{ji}$ |
| Antisimétrica | $A^T = -A$, es decir $a_{ij} = -a_{ji}$ |
| Escalar | diagonal con $a_{ii} = k$ constante |

### Diagonal de matriz cuadrada

La **diagonal principal** es $\{a_{ii}\}$. **Traza**: $\text{tr}(A) = \sum_{i=1}^n a_{ii}$.

### Ley de formación

A menudo se define $A$ mediante la fórmula $a_{ij} = f(i, j)$. Ejemplos:
- $a_{ij} = i + j$
- $a_{ij} = (-1)^{i+j}$
- $a_{ij} = \delta_{ij}$ (delta de Kronecker: genera la identidad)

</Porta>

<Porta nivel="5" titulo="Para un niño de 5 años">

Imagina una **caja de huevos**. Tiene filas y columnas. Cada hueco guarda un huevo (que es un número).

Una **matriz** es solo eso: una caja cuadrada o rectangular llena de números, organizados ordenadamente, cada uno en su sitio. Para decir dónde está un número, dices la fila y la columna: "fila 2, columna 3", listo, lo encuentras.

</Porta>

<Porta nivel="10" titulo="Para un niño de 10 años">

Ya has visto **tablas** en clase. Una tabla de notas: cada alumno es una fila, cada asignatura es una columna. **La matriz es exactamente eso**: una tabla de números.

Ejemplos del día a día:
- **Boletín**: 30 alumnos × 5 asignaturas = matriz 30×5.
- **Pantalla del ordenador**: cada píxel tiene un valor de color. Una pantalla de 1920×1080 es una matriz gigante.
- **Mapa de distancias**: tabla ciudad × ciudad. Fila Madrid, columna Barcelona = 620 km.

Cada número tiene una **dirección**: $a_{ij}$ es "fila $i$, columna $j$". El $i$ va primero, siempre.

Tipos especiales que aparecen mucho:
- **Cuadrada**: número de filas igual al de columnas.
- **Identidad**: igual al número 1: solo tiene 1 en la diagonal y 0 en el resto.
- **Nula**: solo tiene ceros.

</Porta>

<Porta nivel="15" titulo="Para un adolescente de 15 años">

Una matriz es una **tabla** de números organizada en filas y columnas:

$$A = \begin{pmatrix} 1 & 2 & 3 \\ 4 & 5 & 6 \end{pmatrix}$$

Esta es una matriz $2 \times 3$ (2 filas, 3 columnas). Cada número tiene **dirección**: $a_{12} = 2$ (fila 1, columna 2). $a_{23} = 6$ (fila 2, columna 3).

**¿Para qué sirve?** Para guardar y organizar datos, y sobre todo para **hacer cuentas en masa**:
- Notas de los alumnos × asignaturas: matriz alumno-asignatura.
- Distancias entre ciudades: matriz simétrica.
- Píxeles de una imagen: matriz de intensidad.
- Coeficientes de un sistema lineal: matriz de coeficientes.
- Pesos de una red neuronal: vectores y matrices (a toneladas).

**Igualdad**: dos matrices son iguales si tienen **la misma dimensión** Y **todos los elementos correspondientes iguales**. No se puede comparar $2 \times 3$ con $3 \times 2$.

**Tipos importantes**:
- **Cuadrada**: tiene el mismo número de filas y columnas.
- **Identidad** $I_n$: tiene 1 en la diagonal y 0 en el resto. Funciona como el "1" de la multiplicación ($IA = A$).
- **Diagonal**: solo tiene ceros fuera de la diagonal principal.
- **Simétrica**: $A^T = A$: espejo perfecto a través de la diagonal.

**Ley de formación**: se puede construir una matriz a partir de una fórmula. Si $a_{ij} = i + j$, entonces $a_{12} = 3$, $a_{21} = 3$, $a_{33} = 6$, etc.

</Porta>

<Porta nivel="25" titulo="Para un estudiante de ingeniería">

La matriz es el **objeto fundamental** de la computación científica y del ML. En Python/NumPy, "todo es array". En MATLAB, "todo es matriz". El término viene de **Cayley (1858)**, pero la estructura ya estaba implícita en Lagrange y Gauss.

### Visión como aplicación lineal

Una matriz $A_{m \times n}$ representa una aplicación lineal $T: \mathbb{R}^n \to \mathbb{R}^m$ vía $T(\mathbf x) = A\mathbf x$. Las **columnas** son imágenes de los vectores de la base canónica. Esta es la interpretación que hace que las matrices sean omnipresentes.

### Estructuras matriciales comunes

- **Dispersa**: la mayoría de elementos es cero. Los formatos CSR/CSC almacenan solo los no ceros. Crucial en FEM, redes, NLP, grafos con millones de nodos.
- **Banda**: ceros fuera de una franja central. Útil en sistemas tridiagonales (calor, Crank-Nicolson, splines cúbicos).
- **Densa**: todo almacenado. Estándar en ML, álgebra lineal pura, BLAS-3.
- **Block-diagonal**: bloques cuadrados en la diagonal: sistemas desacoplados.
- **Toeplitz**: $a_{ij}$ depende solo de $i - j$: convoluciones, procesamiento de señales.

### Aplicaciones cross-dominio

- **Sistemas de control**: estado $\dot{\mathbf{x}} = A\mathbf{x} + B\mathbf{u}$, $\mathbf y = C\mathbf x$: A es la "dinámica".
- **Estructuras (FEM)**: la matriz de rigidez global $K$ es dispersa, simétrica, definida positiva.
- **Imagen digital**: cada canal RGB es una matriz $H \times W$ de intensidades $\{0, \ldots, 255\}$.
- **Redes neuronales**: cada capa densa es $\mathbf y = \sigma(W\mathbf x + \mathbf b)$.
- **Cuántica**: los estados son vectores, los observables son matrices hermíticas $A = A^*$.
- **Estadística**: la matriz de covarianza $\Sigma$ es simétrica positiva semidefinida.
- **Grafos**: matriz de adyacencia $A_{ij} = 1$ si hay arista: los autovalores codifican la estructura.

### Espacio vectorial $M_{m \times n}(\mathbb R)$

El conjunto de matrices $m \times n$ es un espacio vectorial de **dimensión** $mn$: base canónica $\{E_{ij}\}$ donde $E_{ij}$ tiene 1 en la posición $(i,j)$ y 0 en el resto.

</Porta>

<Porta nivel="40" titulo="Para un profesional sénior">

Las matrices son instancias finitas de **operadores lineales** en espacios vectoriales. La teoría general vive en $\text{Hom}(V, W)$ para $V, W$ espacios vectoriales: fijadas las bases, $\text{Hom}(V, W) \cong M_{m \times n}$.

### Visión estructural

- **$M_n(\mathbb R)$ es un álgebra asociativa no conmutativa** con unidad $I_n$.
- **Descomposiciones espectrales**: SVD ($A = U\Sigma V^T$), autovalores ($A = PDP^{-1}$ si es diagonalizable), Schur, Jordan: cada una revela un aspecto distinto.
- **Grupos de matrices** ($GL_n$, $SO_n$, $U_n$, $Sp_{2n}$) son objetos centrales en geometría, física y teoría de representación.

### Numérico moderno

- **BLAS/LAPACK**: GEMM (multiplicación) es el kernel más optimizado de la computación. $O(n^3)$ teórico, pero memoria/cache dominan: flops "gratis" en GPU.
- **Autodiff y ML**: las bibliotecas (PyTorch, JAX) tratan los tensores como matrices generalizadas. Backprop es la regla de la cadena en grafos de operaciones matriciales.
- **Aleatorización**: random projections (Johnson-Lindenstrauss), sketching, randomized SVD reducen complejidad en big data.

### Quant/Finance

- **Matriz de covarianza** $\Sigma$ de retornos es base de Markowitz. La estimación robusta (shrinkage, Ledoit-Wolf) es problema central.
- **Matrices de transición** en cadenas de Markov modelan ratings de crédito (S&P, Moody's).
- **Cholesky** $\Sigma = LL^T$ genera muestras gaussianas correlacionadas en Monte Carlo.

### Generalizaciones

- **Tensores** ($T_{ijk\ldots}$): generalizan la matriz a múltiples índices. Las descomposiciones (CP, Tucker) son análogos del SVD.
- **Operadores en dimensión infinita**: integrales de Fredholm, operadores compactos: análogos continuos.
- **Matrices sobre cuerpos finitos**: códigos correctores, criptografía (Niederreiter, McEliece), combinatoria.

</Porta>

<Porta nivel="pratica" titulo="Aplicación práctica">

**Workflow para trabajar con matrices en la práctica.**

1. **Identifica la estructura**: ¿dimensión? ¿simétrica? ¿dispersa? ¿triangular? La estructura ahorra memoria y flops.
2. **Elige la representación**: NumPy array (denso), `scipy.sparse` (disperso), `pandas.DataFrame` (con etiquetas), `xarray` (multidim con nombres).
3. **Vectoriza**: nunca hagas bucles `for` para operaciones elemento a elemento; usa broadcasting.

### NumPy — receta

```python
import numpy as np

# Construcción
A = np.array([[1, 2, 3], [4, 5, 6]])     # 2x3 explícita
I = np.eye(3)                             # identidad 3x3
Z = np.zeros((4, 5))                      # nula 4x5
D = np.diag([2, -1, 5])                   # diagonal

# Ley de formación
i, j = np.indices((3, 3))
A_lei = i + j                             # a_ij = i + j

# Acceso
print(A[0, 2])      # elemento fila 0 columna 2 (¡indexación 0-based!)
print(A.shape)      # (2, 3)
print(A.T)          # transpuesta
print(np.trace(np.eye(5)))   # traza
```

### SymPy — simbólico

```python
import sympy as sp
A = sp.Matrix([[1, 2], [3, 4]])
print(A.shape, A.T, A.trace())
```

### Comprobación de simetría

```python
def is_symmetric(A, tol=1e-9):
    return np.allclose(A, A.T, atol=tol)
```

### Trampas comunes

- **Indexación 0-based** en Python/NumPy vs **1-based** en matemáticas. Cuidado al traducir fórmulas.
- **Fila vs columna**: `A[i]` es la fila $i$. Para columna usa `A[:, j]`.
- **Copia vs view**: `B = A` ¡no copia! Usa `B = A.copy()`.
- **`np.matrix` está deprecated**. Usa `np.ndarray` (`@` para multiplicación matricial).
