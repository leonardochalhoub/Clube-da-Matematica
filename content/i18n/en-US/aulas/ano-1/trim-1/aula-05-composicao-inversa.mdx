<Porta nivel="formal" titulo="Composition and Inverse">

<Definicao titulo="Composition">
Given $f: B \to C$ and $g: A \to B$, the composition $f \circ g: A \to C$ is defined by $(f \circ g)(x) = f(g(x))$.
</Definicao>

<Definicao titulo="Inverse function">
$f: A \to B$ admits an **inverse** if and only if $f$ is bijective. In this case, $f^{-1}: B \to A$ satisfies:
$$f^{-1}(f(x)) = x \quad \forall x \in A, \qquad f(f^{-1}(y)) = y \quad \forall y \in B$$
</Definicao>

<Porta nivel="5" titulo="For a 5-year-old">

You have **two machines**.

The first machine takes a number and **adds 1**. Put 3, out comes 4. Put 7, out comes 8.

The second machine takes a number and **multiplies by 2**. Put 4, out comes 8. Put 5, out comes 10.

Now connect both in sequence: what comes out of the first **goes straight into the second**. Put 3 in the first → out comes 4 → into the second → out comes 8.

This is **composition** of functions: using two in a row.

And **inverse** is a machine that undoes another. If a machine adds 1, the inverse subtracts 1. If you add and then the inverse subtracts, you go back to the original number.

</Porta>

<Porta nivel="10" titulo="For a 10-year-old">

You have a formula: "take the number, multiply by 3, then add 5". In math: $f(x) = 3x + 5$.

**Composition** is doing two formulas in order. For example, if $g(x) = x - 2$ and $f(x) = 3x + 5$, then $f$ composed with $g$, written $(f \circ g)(x)$, is:

1. Apply $g$ to $x$: gives $x - 2$.
2. Take what's left, throw it into $f$: gives $3(x-2) + 5 = 3x - 1$.

**Inverse** is "undoing". If the formula is $f(x) = 2x + 3$, to undo you reverse each operation in the opposite order: subtract 3, then divide by 2. Result: $f^{-1}(x) = (x - 3)/2$.

**Visual trick.** On the graph, the inverse is the reflection of the original about the **diagonal line** $y = x$. It's like looking at the function in a mirror.

**Catch.** Not every function has an inverse. $f(x) = x^2$ is a function that does NOT have an inverse — because $f(2) = 4$ and $f(-2) = 4$ as well. If I give you only "4", you cannot know if I put 2 or −2 in the machine. Without enough information, you can't undo.

</Porta>

<Porta nivel="15" titulo="For a 15-year-old teen">

**Composition** is doing two operations in sequence. "Take a number, multiply by 2, then add 1": composition of $g(x) = 2x$ with $h(y) = y + 1$ — written $h(g(x)) = h(2x) = 2x + 1$.

**Inverse** is undoing. If $f(x) = 2x + 1$ takes 3 and returns 7, the inverse $f^{-1}$ takes 7 and returns 3.

To find the inverse: swap $x$ for $y$ in the equation, and isolate $y$.

$$y = 2x + 1 \quad \to \quad x = 2y + 1 \quad \to \quad y = \frac{x - 1}{2}$$

Therefore $f^{-1}(x) = (x-1)/2$. Check: $f^{-1}(7) = 6/2 = 3$. ✓

**Caution**: not every function has an inverse. $f(x) = x^2$ doesn't, because $f(2) = f(-2) = 4$ — you can't tell whether the "original input" was 2 or −2.

</Porta>

<Porta nivel="25" titulo="For an engineering student">

Composition is fundamental — every "complicated" function decomposes into compositions of "simple" functions. This decomposition is what makes the **chain rule** (Term 6) computable.

### Inverse and bijection

$f$ has an inverse $\iff$ $f$ is bijective. In practice:
- **Not injective?** Restrict the domain. $f(x) = x^2$ becomes injective on $[0, +\infty)$, with inverse $\sqrt{y}$.
- **Not surjective?** Restrict the codomain to the image.

### Applications

- **Cryptography**: $E$ encrypts, $D$ decrypts. $D = E^{-1}$. Without an inverse, no decryption.
- **Control engineering**: input-output relation and plant identification.
- **Thermodynamics**: $P-V$, $T-S$ relations, etc., need to be injective for invertibility.

### Notation

$f^{-1}$ is NOT $1/f$. Students confuse this even into the 2nd year of college. Be careful.

### Operator identity

The identity function $I(x) = x$ is the **neutral element** of composition: $f \circ I = I \circ f = f$. The inverse is the **inverse** of this operation: $f \circ f^{-1} = f^{-1} \circ f = I$. This makes the bijective functions from $A$ to $A$ a **group** under composition — called the **symmetric group** $\text{Sym}(A)$ when $A$ is finite. It's the first algebraic structure you'll see in abstract algebra in college.

### Composition preserves properties

If $f$ and $g$ are continuous, $f \circ g$ is continuous. If both are differentiable, $f \circ g$ is differentiable (chain rule). But: composition of **convex** functions is not always convex — it depends on signs and monotonicities.

</Porta>

<Porta nivel="40" titulo="For a senior professional">

Composition is the **basic operator** of functional programming. In Haskell, `(f . g) x = f (g x)`. In Python (functional programming), `compose(f, g)(x) = f(g(x))`. Every data transformation is a chain of compositions — Pandas `.pipe`, dplyr `%>%`, sklearn `Pipeline`.

### Inverse in ill-posed problems

In inverse engineering (tomography, seismology, inverse control), the problem is: **given $y$, find $x$ such that $f(x) = y$**. If $f$ is not injective or if measurements have noise, "$f^{-1}$" is ill-defined. Dealing with this is **regularization** — Tikhonov, ridge, lasso. All statistical inference is a form of regularized inversion.

### Inverse of linear transformations

For $f(x) = Ax$ (matrix form), $f$ is invertible $\iff$ $A$ is invertible $\iff \det A \neq 0$ $\iff$ all eigenvalues are non-zero. This will be explored in Lessons 33-35 (Term 4) and 111-120 (Year 3 — Term 12).

### Iterated functions and dynamics

$f^n(x) = f(f(\ldots f(x)\ldots))$ ($n$ times). One studies orbits, fixed points, basins of attraction — the basis of **dynamical systems**. The Mandelbrot set is defined by the divergence of iteration. In ML, **deep networks** are compositions $f_L \circ f_{L-1} \circ \ldots \circ f_1$ — every neural network is a composed function.

### Caution: inverse at industrial scale

Computing $f^{-1}$ analytically is rare. In production, **bisection** or **Newton-Raphson** is used to solve $f(x) = y$ for each $y$ (see Numerical Methods in Term 10). For example, computing implied volatility from Black-Scholes (page `/financas/.../black-scholes`) is exactly inverting the BS formula — done numerically, not analytically.

</Porta>

<Porta nivel="pratica" titulo="Practice — 3 real scenarios">

### Scenario 1: Data transformation pipeline (sklearn-style)

You have height data in centimeters and want to transform to z-scores (mean 0, std 1):

1. $g(x) = x - \bar x$ (centering)
2. $f(y) = y / \sigma$ (scaling)

Composition: $(f \circ g)(x) = (x - \bar x)/\sigma$. Inverse (for un-transforming predictions): $g^{-1}(z) = z + \bar x$, $f^{-1}(z) = z \cdot \sigma$. Composition inverse: $(f \circ g)^{-1}(z) = (g^{-1} \circ f^{-1})(z) = z \cdot \sigma + \bar x$. **Note the order reversal!**

### Scenario 2: Thermometric conversion (composition in series)

Celsius → Kelvin conversion: $K(C) = C + 273.15$.
Kelvin → Rankine conversion: $R(K) = K \cdot 1.8$.
Composition C → R: $(R \circ K)(C) = (C + 273.15) \cdot 1.8 = 1.8 C + 491.67$.

Inverse Rankine → Celsius: $(R \circ K)^{-1}(r) = (r - 491.67)/1.8$.

### Scenario 3: Password hash (cryptography)

Your bank password goes through a **hash** function $h$ (SHA-256, bcrypt, argon2). $h$ is deliberately designed to **NOT have a computable inverse**. You can compute $h(\text{password})$ quickly, but given the hash $y$, finding $\text{password}$ with $h(\text{password}) = y$ is computationally infeasible. This **invertibility asymmetry** is the basis of all modern authentication. If $h$ had an easy inverse, any database leak would be fatal.

</Porta>
