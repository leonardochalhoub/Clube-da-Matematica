<Porta nivel="formal" titulo="Rigorous definition">

<Definicao titulo="Function">

Let $A$ and $B$ be non-empty sets. A **function** $f$ from $A$ to $B$, denoted $f: A \to B$, is a relation that assigns to each $x \in A$ a unique element $f(x) \in B$.

- $A$ is the **domain** of $f$, denoted $\operatorname{dom}(f)$.
- $B$ is the **codomain**.
- $\operatorname{Im}(f) = \{f(x) : x \in A\} \subseteq B$ is the **image**.

</Definicao>

### Classification

<Definicao titulo="Types of function">

- **Injective** (1-1): $x_1 \neq x_2 \Rightarrow f(x_1) \neq f(x_2)$.
- **Surjective**: $\operatorname{Im}(f) = B$ (every $y \in B$ has at least one $x$ mapping to it).
- **Bijective**: injective AND surjective.

</Definicao>

<Porta nivel="5" titulo="For a 5-year-old">

Imagine a magical machine. You put a **number** inside. The machine thinks for a little bit. And another **number** comes out.

Every time you put in the same number, the machine always gives the same answer. Always. If the machine sometimes gives one answer and sometimes another, it isn't a "function" — it's just a crazy machine.

</Porta>

<Porta nivel="10" titulo="For a 10-year-old">

A function is a **fixed relation** between two numbers.

Example: doubling a number. If **3** goes in, **6** comes out. If **5** goes in, **10** comes out. If **8.7** goes in, **17.4** comes out. The rule is "output = input × 2".

We write this as $f(x) = 2x$. The **f** is just a name — you could call it **g**, **h**, **double**, anything. The **(x)** is the place where you put the number that will go in.

Functions show up in **EVERYTHING** in science:
- A car's speed as a **function of time**: $v(t) = ?$
- The cost of a trip as a **function of distance**: $C(d) = ?$
- Temperature throughout the day as a **function of the hour**: $T(h) = ?$

</Porta>

<Porta nivel="15" titulo="For a 15-year-old">

A function $f$ is a rule that associates each input with a single output. The set of valid inputs is the **domain**, and the set of outputs that actually appear is the **image**.

Example: $f(x) = x^2$.

| $x$ (input) | $f(x)$ (output) |
|---:|---:|
| −2 | 4 |
| −1 | 1 |
| 0 | 0 |
| 1 | 1 |
| 2 | 4 |

Domain: $\mathbb{R}$ (any real serves as input). Image: $[0, +\infty)$ (output is never negative).

**The graph** of this function is a parabola:

<Porta nivel="25" titulo="For an engineering student">

The function is the most important mathematical object of the entire calculus curriculum. Before differentiating or integrating, you need absolute clarity on what a function is.

### Maximum domain

When the problem says "study $f(x) = \sqrt{x - 1}$" without specifying a domain, we adopt the **natural maximum domain**: the largest subset of $\mathbb{R}$ on which the expression makes sense.

- $\sqrt{x-1}$: $x - 1 \geq 0 \Rightarrow x \geq 1$, domain = $[1, +\infty)$.
- $1/(x-2)$: $x \neq 2$, domain = $\mathbb{R} \setminus \{2\}$.
- $\ln(x)$: $x > 0$, domain = $(0, +\infty)$.

### Composition

$(f \circ g)(x) = f(g(x))$. **Caution**: composition is not commutative in general.

Example: $f(x) = x^2$, $g(x) = x + 1$.
- $(f \circ g)(x) = f(x+1) = (x+1)^2$
- $(g \circ f)(x) = g(x^2) = x^2 + 1$

Different!

### Why injectivity matters

A function only admits an **inverse** if it is injective. Non-injective functions need to be restricted. Classic example: $f(x) = x^2$ is not injective on $\mathbb{R}$, but it is injective on $[0, +\infty)$, where its inverse is $f^{-1}(y) = \sqrt{y}$.

### Functions and modeling

In engineering, a function is the object that represents a **phenomenon**:
- $T(t)$: temperature as a function of time (sensor)
- $V(I)$: voltage as a function of current (Ohm's law)
- $\sigma(\eps)$: stress as a function of strain (Hooke's law)

Every dimensional analysis, every calibration, every model fit begins by understanding what the function is.

</Porta>

<Porta nivel="40" titulo="For a senior professional">

The definition of function as "rule that assigns" is informal. The rigorous definition, à la Bourbaki, is:

> A function $f: A \to B$ is a **subset** $f \subseteq A \times B$ such that for each $x \in A$ there exists **exactly one** $y \in B$ with $(x, y) \in f$. We write $y = f(x)$.

That is, a function IS its graph (in the set-theoretic sense). Operational definition for Logic and Set Theory. Day-to-day calculus does not use this formalization, but you'll see it in Real Analysis (Calculus IV / university Real Analysis).

### Functions as morphisms

In Categories (modern language of mathematics), functions are **morphisms** between sets. A bijection is an isomorphism in the category $\mathbf{Set}$. Composition of functions is composition of morphisms.

### Functions and computing

In computer science, function is the central concept of functional programming (Haskell, Lisp). Currying — turning $f: A \times B \to C$ into $f': A \to (B \to C)$ — is routine type manipulation. A "real" everyday production function may have a signature like `User -> Order -> Promise<Receipt>` — which is exactly a mathematical $f: A \to B \to C$ with side effects.

### Modeling limitations

In real physical phenomena, "function" is an abstraction:
- **Hysteresis**: state depends not only on current input but on past trajectory (plastic deformation). Not a function.
- **Quantum**: measurement is probabilistic — there is no single deterministic value. It is a probability distribution.
- **Chaos**: small variations in initial conditions generate arbitrarily different trajectories. Deterministic in theory, unpredictable in practice.

In all these cases, the function concept is insufficient, and richer abstractions (maps, stochastic processes, dynamics) come into play.

</Porta>

<Porta nivel="pratica" titulo="Practice — examples with real data">

### Example 1: Taxi fare (affine model)

In São Paulo, a taxi charges:

$$T(d) = 5{,}50 + 3{,}10 \cdot d$$

where $T$ is the value in R\$ and $d$ the distance in km.

- Domain: $d \geq 0$ → $[0, +\infty)$
- Image: $T \geq 5{,}50$ → $[5{,}50, +\infty)$
- $T(0) = 5{,}50$ is the **flag-fall** (fixed boarding cost)
- $T(10) = 5{,}50 + 31 = 36{,}50$

### Example 2: BMI (Body Mass Index)

$$\text{BMI}(p, h) = \frac{p}{h^2}$$

- Function of **two variables** (weight $p$ in kg, height $h$ in meters).
- Physical domain: $p > 0$, $h > 0$ → $(0, +\infty)^2$.
- Image: $(0, +\infty)$.

WHO classifies:
- BMI < 18.5: underweight
- 18.5 ≤ BMI < 25: normal
- 25 ≤ BMI < 30: overweight
- BMI ≥ 30: obesity

### Example 3: Selic rate (function of time, discontinuous)

The Selic is decided by Copom in meetings every ~45 days. Between meetings, it is constant. "Step" function:

```
Selic(jan/2025) = 13.25%
Selic(mar/2025) = 14.25%
Selic(may/2025) = 14.75%  (held since then)
```

Domain: time (dates). Codomain: percentages. Image: discrete, values dictated by Copom.

</Porta>
