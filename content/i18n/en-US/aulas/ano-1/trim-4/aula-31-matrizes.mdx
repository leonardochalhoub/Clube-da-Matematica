<Porta nivel="formal" titulo="Definition and types">

<Definicao titulo="Matrix">
An $m \times n$ matrix over $\mathbb{R}$ is a function $A: \{1, \ldots, m\} \times \{1, \ldots, n\} \to \mathbb{R}$. Notation $A = (a_{ij})$ or $A_{m \times n}$.

Equality: $A = B \iff$ same dimension and $a_{ij} = b_{ij}$ for all $i, j$.
</Definicao>

### Special types

| Type | Definition |
|---|---|
| Square | $m = n$ |
| Row | $1 \times n$ |
| Column | $m \times 1$ |
| Diagonal | square with $a_{ij} = 0$ for $i \neq j$ |
| Identity $I_n$ | diagonal with $a_{ii} = 1$ |
| Zero $O$ | all entries zero |
| Upper triangular | $a_{ij} = 0$ for $i > j$ |
| Lower triangular | $a_{ij} = 0$ for $i < j$ |
| Symmetric | $A^T = A$, that is $a_{ij} = a_{ji}$ |
| Skew-symmetric | $A^T = -A$, that is $a_{ij} = -a_{ji}$ |
| Scalar | diagonal with $a_{ii} = k$ constant |

### Diagonal of a square matrix

The **main diagonal** is $\{a_{ii}\}$. **Trace**: $\text{tr}(A) = \sum_{i=1}^n a_{ii}$.

### Construction rule

Often $A$ is defined via a formula $a_{ij} = f(i, j)$. Examples:
- $a_{ij} = i + j$
- $a_{ij} = (-1)^{i+j}$
- $a_{ij} = \delta_{ij}$ (Kronecker delta — produces the identity)

</Porta>

<Porta nivel="5" titulo="For a 5-year-old">

Imagine an **egg carton**. It has rows and columns. Each spot holds an egg (which is a number).

A **matrix** is just that: a square or rectangular box full of numbers, neatly organized — each one in its place. To say where a number is, you say the row and the column: "row 2, column 3" — there it is.

</Porta>

<Porta nivel="10" titulo="For a 10-year-old">

You've seen **tables** in class. A grade table: each student is a row, each subject is a column. **A matrix is exactly that** — a table of numbers.

Examples from daily life:
- **Report card**: 30 students × 5 subjects = a 30×5 matrix.
- **Computer screen**: each pixel has a color value. A 1920×1080 screen is a giant matrix.
- **Distance map**: city × city table. Row São Paulo, column Rio = 430 km.

Each number has an **address**: $a_{ij}$ is "row $i$, column $j$". $i$ comes first, always.

Special types that show up a lot:
- **Square**: same number of rows and columns.
- **Identity**: like the number 1 — has 1 on the diagonal and 0 everywhere else.
- **Zero**: only zeros.

</Porta>

<Porta nivel="15" titulo="For a 15-year-old teen">

A matrix is a **table** of numbers organized in rows and columns:

$$A = \begin{pmatrix} 1 & 2 & 3 \\ 4 & 5 & 6 \end{pmatrix}$$

This is a $2 \times 3$ matrix (2 rows, 3 columns). Each number has an **address**: $a_{12} = 2$ (row 1, column 2). $a_{23} = 6$ (row 2, column 3).

**What's it for?** To store and organize data, and especially to **do bulk computations**:
- Student grades × subjects: student-subject matrix.
- Distances between cities: symmetric matrix.
- Image pixels: intensity matrix.
- Coefficients of a linear system: coefficient matrix.
- Weights of a neural network: vectors and matrices (tons).

**Equality**: two matrices are equal if they have the **same dimension** AND **all corresponding entries equal**. You cannot compare $2 \times 3$ with $3 \times 2$.

**Important types**:
- **Square**: same number of rows and columns.
- **Identity** $I_n$: 1's on the diagonal and 0 everywhere else. It works as the "1" of multiplication ($IA = A$).
- **Diagonal**: only zeros off the main diagonal.
- **Symmetric**: $A^T = A$ — perfect mirror across the diagonal.

**Construction rule**: you can build a matrix from a formula. If $a_{ij} = i + j$, then $a_{12} = 3$, $a_{21} = 3$, $a_{33} = 6$, etc.

</Porta>

<Porta nivel="25" titulo="For an engineering student">

A matrix is the **fundamental object** of scientific computing and ML. In Python/NumPy, "everything is an array". In MATLAB, "everything is a matrix". The term is from **Cayley (1858)**, but the structure was already implicit in Lagrange and Gauss.

### View as a linear map

A matrix $A_{m \times n}$ represents a linear map $T: \mathbb{R}^n \to \mathbb{R}^m$ via $T(\mathbf x) = A\mathbf x$. The **columns** are images of canonical basis vectors. This is the interpretation that makes matrices ubiquitous.

### Common matrix structures

- **Sparse**: most entries are zero. CSR/CSC formats store only non-zeros. Crucial in FEM, networks, NLP, graphs with millions of nodes.
- **Banded**: zeros outside a central band. Useful in tridiagonal systems (heat, Crank-Nicolson, cubic splines).
- **Dense**: everything stored. Standard in ML, pure linear algebra, BLAS-3.
- **Block-diagonal**: square blocks on the diagonal — uncoupled systems.
- **Toeplitz**: $a_{ij}$ depends only on $i - j$ — convolutions, signal processing.

### Cross-domain applications

- **Control systems**: state $\dot{\mathbf{x}} = A\mathbf{x} + B\mathbf{u}$, $\mathbf y = C\mathbf x$ — A is the "dynamics".
- **Structures (FEM)**: global stiffness matrix $K$ is sparse, symmetric, positive-definite.
- **Digital image**: each RGB channel is an $H \times W$ matrix of intensities $\{0, \ldots, 255\}$.
- **Neural networks**: each dense layer is $\mathbf y = \sigma(W\mathbf x + \mathbf b)$.
- **Quantum**: states are vectors, observables are Hermitian matrices $A = A^*$.
- **Statistics**: covariance matrix $\Sigma$ is symmetric positive semidefinite.
- **Graphs**: adjacency matrix $A_{ij} = 1$ if there is an edge — eigenvalues encode structure.

### Vector space $M_{m \times n}(\mathbb R)$

The set of $m \times n$ matrices is a vector space of **dimension** $mn$ — canonical basis $\{E_{ij}\}$ where $E_{ij}$ has 1 at position $(i,j)$ and 0 elsewhere.

</Porta>

<Porta nivel="40" titulo="For a senior professional">

Matrices are finite instances of **linear operators** on vector spaces. The general theory lives in $\text{Hom}(V, W)$ for $V, W$ vector spaces — once bases are fixed, $\text{Hom}(V, W) \cong M_{m \times n}$.

### Structural view

- **$M_n(\mathbb R)$ is an associative non-commutative algebra** with identity $I_n$.
- **Spectral decompositions**: SVD ($A = U\Sigma V^T$), eigenvalues ($A = PDP^{-1}$ if diagonalizable), Schur, Jordan — each reveals a distinct aspect.
- **Matrix groups** ($GL_n$, $SO_n$, $U_n$, $Sp_{2n}$) are central objects in geometry, physics, and representation theory.

### Modern numerics

- **BLAS/LAPACK**: GEMM (multiplication) is the most optimized kernel in computing. Theoretical $O(n^3)$, but memory/cache dominate — flops are "free" on the GPU.
- **Autodiff and ML**: libraries (PyTorch, JAX) treat tensors as generalized matrices. Backprop is the chain rule on graphs of matrix operations.
- **Randomization**: random projections (Johnson-Lindenstrauss), sketching, randomized SVD reduce complexity in big data.

### Quant/Finance

- **Covariance matrix** $\Sigma$ of returns is the basis of Markowitz. Robust estimation (shrinkage, Ledoit-Wolf) is a central problem.
- **Transition matrices** in Markov chains model credit ratings (S&P, Moody's).
- **Cholesky** $\Sigma = LL^T$ generates correlated Gaussian samples in Monte Carlo.

### Generalizations

- **Tensors** ($T_{ijk\ldots}$): generalize matrices to multiple indices. Decompositions (CP, Tucker) are SVD analogs.
- **Operators in infinite dim**: Fredholm integrals, compact operators — continuous analogs.
- **Matrices over finite fields**: error-correcting codes, cryptography (Niederreiter, McEliece), combinatorics.

</Porta>

<Porta nivel="pratica" titulo="Practical application">

**Workflow for working with matrices in practice.**

1. **Identify the structure**: dimension? symmetric? sparse? triangular? Structure saves memory and flops.
2. **Choose the representation**: NumPy array (dense), `scipy.sparse` (sparse), `pandas.DataFrame` (with labels), `xarray` (multi-dim with names).
3. **Vectorize**: never use `for` loops for element-wise operations — use broadcasting.

### NumPy — recipe

```python
import numpy as np

# Construction
A = np.array([[1, 2, 3], [4, 5, 6]])     # explicit 2x3
I = np.eye(3)                             # 3x3 identity
Z = np.zeros((4, 5))                      # 4x5 zero
D = np.diag([2, -1, 5])                   # diagonal

# Construction rule
i, j = np.indices((3, 3))
A_rule = i + j                            # a_ij = i + j

# Access
print(A[0, 2])      # row 0 column 2 (0-based indexing!)
print(A.shape)      # (2, 3)
print(A.T)          # transpose
print(np.trace(np.eye(5)))   # trace
```

### SymPy — symbolic

```python
import sympy as sp
A = sp.Matrix([[1, 2], [3, 4]])
print(A.shape, A.T, A.trace())
```

### Symmetry check

```python
def is_symmetric(A, tol=1e-9):
    return np.allclose(A, A.T, atol=tol)
```

### Common pitfalls

- **0-based indexing** in Python/NumPy vs **1-based** in math. Be careful when translating formulas.
- **Row vs column**: `A[i]` is row $i$. For column use `A[:, j]`.
- **Copy vs view**: `B = A` does not copy! Use `B = A.copy()`.
- **`np.matrix` is deprecated**. Use `np.ndarray` (`@` for matrix multiplication).
