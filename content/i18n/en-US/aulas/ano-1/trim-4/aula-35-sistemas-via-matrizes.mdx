<Porta nivel="formal" titulo="Solution methods">

### Matrix form

$\begin{cases} a_1 x + b_1 y + c_1 z = d_1 \\ a_2 x + b_2 y + c_2 z = d_2 \\ a_3 x + b_3 y + c_3 z = d_3 \end{cases}$ ⟺ $A\mathbf{x} = \mathbf{b}$ with $A_{3 \times 3}$, $\mathbf{x}, \mathbf{b} \in \mathbb{R}^3$.

### Method 1 — Gaussian elimination

Elementary operations (do not change solution):
1. Swap two rows.
2. Multiply a row by a non-zero scalar.
3. Add a multiple of one row to another.

Goal: triangularize $[A | \mathbf{b}]$ to row-echelon form. Then back-substitute.

### Method 2 — Cramer

For $A\mathbf{x} = \mathbf{b}$ with $\det A \neq 0$:
$$x_i = \frac{\det A_i}{\det A}$$

where $A_i$ is $A$ with the $i$-th column replaced by $\mathbf{b}$.

### Method 3 — Inverse

$\mathbf{x} = A^{-1} \mathbf{b}$. $A^{-1}$ can be computed via $[A | I] \to [I | A^{-1}]$ by row reduction.

### When to use each

| Method | Cost | When to use |
|---|---|---|
| Cramer | $O(n^4)$ | $n \leq 3$, didactic |
| Gauss | $O(n^3)$ | practical default |
| Inverse | $O(n^3)$ + $O(n^2)$/system | many $\mathbf b$ with same $A$ |
| LU | $O(n^3)$ + $O(n^2)$/system | many systems, better than inverse |

### Classification of systems

- **Determinate (unique)**: unique solution ($\det A \neq 0$, or $\text{rank}(A) = \text{rank}([A|\mathbf b]) = n$).
- **Indeterminate**: infinitely many solutions ($\text{rank}(A) = \text{rank}([A|\mathbf b]) < n$).
- **Impossible**: no solution ($\text{rank}(A) < \text{rank}([A|\mathbf b])$).

### Rouché-Capelli theorem

$A\mathbf x = \mathbf b$ has a solution ⟺ $\text{rank}(A) = \text{rank}([A|\mathbf b])$. Solution is unique if both equal the number of unknowns.

</Porta>

<Porta nivel="5" titulo="For a 5-year-old">

A system is **a bunch of equations at the same time**. Like: "I have 3 apples and 2 pears, total 13 dollars. I have 1 apple and 4 pears, total 16 dollars. How much does each one cost?"

To solve, you **combine** the equations cleverly to **eliminate** one of the fruits, find the price of the other, and go back. It's a puzzle with a fixed recipe.

</Porta>

<Porta nivel="10" titulo="For a 10-year-old">

**Linear system** = several equations with several unknowns that need to be true at the same time.

Example:
$$\begin{cases} x + y = 5 \\ x - y = 1 \end{cases}$$

How to solve?
- **Add the two**: $2x = 6 \Rightarrow x = 3$.
- **Subtract**: $2y = 4 \Rightarrow y = 2$.

Done: $x = 3, y = 2$.

**For large systems**, doing it by hand is tedious. So we **organize into a matrix**:
$$\begin{pmatrix} 1 & 1 \\ 1 & -1 \end{pmatrix} \begin{pmatrix} x \\ y \end{pmatrix} = \begin{pmatrix} 5 \\ 1 \end{pmatrix}$$

And use **row reduction** (standard) or **Cramer's rule** (with determinants) or **inverse** ($\mathbf x = A^{-1}\mathbf b$).

**When does it fail?**
- System with **no solution**: equations contradict each other.
- System with **infinitely many solutions**: equations are redundant.

</Porta>

<Porta nivel="15" titulo="For a 15-year-old teen">

Linear system in matrix form: pack the coefficients in $A$, the unknowns in $\mathbf{x}$, and the results in $\mathbf{b}$. The system becomes $A\mathbf{x} = \mathbf{b}$.

**Cramer** (didactic, small): $x_i = \det(A_i)/\det(A)$. Example:
$$\begin{cases} 2x + y = 5 \\ x - 3y = -2 \end{cases} \Rightarrow A = \begin{pmatrix} 2 & 1 \\ 1 & -3 \end{pmatrix}, \det A = -7$$

$x = \det\begin{pmatrix} 5 & 1 \\ -2 & -3 \end{pmatrix}/(-7) = (-13)/(-7) = 13/7$.

$y = \det\begin{pmatrix} 2 & 5 \\ 1 & -2 \end{pmatrix}/(-7) = (-9)/(-7) = 9/7$.

**Gauss** (standard): perform elementary operations to triangularize. Then back-substitute from bottom to top.

$$[A | \mathbf b] = \begin{pmatrix} 2 & 1 & | & 5 \\ 1 & 3 & | & 6 \end{pmatrix} \xrightarrow{L_2 \leftarrow L_2 - L_1/2} \begin{pmatrix} 2 & 1 & | & 5 \\ 0 & 5/2 & | & 7/2 \end{pmatrix}$$

From the second row: $y = 7/5$. Substitute into the first: $2x = 5 - 7/5 \Rightarrow x = 9/5$.

**Classification**:
- $\det A \neq 0$ → **determinate** (unique solution).
- $\det A = 0$ + consistent → **indeterminate** (infinitely many).
- $\det A = 0$ + inconsistent → **impossible**.

**Homogeneous system** $A\mathbf x = \mathbf 0$: always has $\mathbf x = \mathbf 0$ (trivial). Has a non-trivial solution ⟺ $\det A = 0$.

</Porta>

<Porta nivel="25" titulo="For an engineering student">

Solving $A\mathbf x = \mathbf b$ is the **most common** operation in all scientific computing. ~80% of FEM/CFD simulation time is spent in linear solvers.

### Solver hierarchy

1. **Direct methods** (Gauss, LU, QR, Cholesky): $O(n^3)$, exact up to machine error.
2. **Iterative methods** (Jacobi, Gauss-Seidel, CG, GMRES, BiCGSTAB): $O(n^2)$ per iter, requires convergence, ideal for sparse.
3. **Specialized methods**: FFT for circulants, multigrid for PDEs.

### LU with partial pivoting

$PA = LU$ — $P$ permutation, $L$ unit lower-triangular, $U$ upper-triangular. Solves $A\mathbf x = \mathbf b$ via:
1. $L\mathbf y = P\mathbf b$ (forward sub, $O(n^2)$).
2. $U\mathbf x = \mathbf y$ (back sub, $O(n^2)$).

Pivoting is crucial for stability — avoids division by small numbers.

### Ill-conditioned systems

$\kappa(A) = \|A\|\|A^{-1}\|$. For Gauss in float64, lose $\log_{10}\kappa$ digits. If $\kappa = 10^{16}$, the solution is pure garbage.

### Least squares (overdetermined system)

$\min_\mathbf x \|A\mathbf x - \mathbf b\|^2$. Normal equation: $A^TA\mathbf x = A^T\mathbf b$. **Don't use!** Conditioning squared. Use QR: $A = QR$ ⟹ $R\mathbf x = Q^T\mathbf b$.

### Cross-domain

- **Circuits**: KCL/KVL produce linear systems — Modified Nodal Analysis.
- **FEM**: $K\mathbf u = \mathbf f$ — millions of unknowns, sparse, symmetric positive-definite.
- **Optimization**: each Newton iteration solves $H\Delta\mathbf x = -\nabla f$.
- **Statistics**: regression $X^TX\hat\beta = X^T\mathbf y$ (but use QR/SVD).
- **Filters (Kalman)**: predict-correct involves solvers each step.

### Numerical — best practices

- **Use `solve`, not `inv`**.
- For SPD: Cholesky.
- For sparse: `scipy.sparse.linalg.spsolve` or iterative with preconditioner.
- For dense $> 10^4$: GPU or iterative methods.

</Porta>

<Porta nivel="40" titulo="For a senior professional">

Linear solvers are the **basis** of PDEs, ML, optimization, and simulation. Method choice is an architectural decision — it can be the difference between a simulation being feasible or not.

### Wolfram tour

```mathematica

```

### Matrix classes and optimal algorithm

| Type | Solver |
|---|---|
| Dense general | LU (Gauss + pivot) |
| Dense SPD | Cholesky |
| Dense rectangular | QR |
| Sparse symmetric indef. | MINRES, SYMMLQ |
| Sparse SPD | CG (with precond.) |
| Sparse non-symmetric | GMRES, BiCGSTAB |
| Structured (Toeplitz) | FFT-based, $O(n \log n)$ |
| Tridiagonal | Thomas, $O(n)$ |
| Banded | banded LU $O(nb^2)$ |

### Modern iteratives

- **Krylov subspace**: $\mathcal K_k = \text{span}\{\mathbf b, A\mathbf b, \ldots, A^{k-1}\mathbf b\}$. CG (SPD), GMRES (general) build optimal approximations in $\mathcal K_k$.
- **Convergence**: $O(\sqrt{\kappa})$ iterations for CG. Preconditioning $\widetilde A = M^{-1}A$ with $\kappa(\widetilde A) \ll \kappa(A)$ is an art.

### Big data / sparse

- **Hypre, PETSc, Trilinos**: industrial libraries for systems $> 10^9$.
- **Multigrid**: $O(n)$ for elliptic problems.
- **Domain decomposition**: parallelizes by splitting $\Omega$ into subdomains.

### Quant — risk systems

Calibration of models (BS, Heston, SABR) creates ill-conditioned systems. **Levenberg-Marquardt**, **Tikhonov regularization**, **truncated SVD** are standard.

### Current research

- **Randomized numerical linear algebra**: sketching reduces dim before solving. Drineas, Mahoney.
- **Mixed precision iterative refinement**: solve in FP32, refine in FP64. 10× gain on modern hardware.
- **Quantum algorithms (HHL)**: $O(\log n)$ in qubits for certain classes — still theoretical.

</Porta>

<Porta nivel="pratica" titulo="Practical application">

**Workflow for solving $A\mathbf x = \mathbf b$ in practice.**

1. Look at the structure: symmetric? sparse? triangular?
2. Choose the compatible solver.
3. **Never use the explicit inverse** if the goal is to solve.
4. Check conditioning; always verify the residual $\|A\mathbf x - \mathbf b\|$.

### NumPy recipe

```python
import numpy as np
from scipy.linalg import lu, cho_factor, cho_solve, qr, solve_triangular

A = np.array([[2, 1], [1, 3]], dtype=float)
b = np.array([5, 7], dtype=float)

# Standard
x = np.linalg.solve(A, b)

# For SPD
c, lo = cho_factor(A)
x = cho_solve((c, lo), b)

# Least squares (overdetermined)
A_rect = np.random.randn(100, 5)
b_rect = np.random.randn(100)
x_lsq, residuals, rank, sv = np.linalg.lstsq(A_rect, b_rect, rcond=None)

# Sparse
from scipy.sparse import csr_matrix
from scipy.sparse.linalg import spsolve
A_sp = csr_matrix(A)
x_sp = spsolve(A_sp, b)

# Iterative (CG for SPD)
from scipy.sparse.linalg import cg
x_cg, info = cg(A_sp, b)
```

### Symbolic SymPy

```python
import sympy as sp
A = sp.Matrix([[2, 1], [1, 3]])
b = sp.Matrix([5, 7])
x = A.solve(b)             # exact in fractions
print(x)
```

### Manual 2×2 Cramer

```python
def cramer_2x2(A, b):
    detA = A[0,0]*A[1,1] - A[0,1]*A[1,0]
    if abs(detA) < 1e-14:
        return None
    Ax = np.array([[b[0], A[0,1]], [b[1], A[1,1]]])
    Ay = np.array([[A[0,0], b[0]], [A[1,0], b[1]]])
    return np.array([np.linalg.det(Ax)/detA, np.linalg.det(Ay)/detA])
```

### Pitfalls

- If `solve` raises `LinAlgError: singular matrix`: use `lstsq` or `pinv`.
- A solution can exist even with $\det \approx 0$ — it depends on $\mathbf b$ being in the column space.
- **Always verify** the residual: `np.linalg.norm(A @ x - b)`.
