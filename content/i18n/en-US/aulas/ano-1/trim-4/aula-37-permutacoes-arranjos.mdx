<Porta nivel="formal" titulo="Definitions">

### Factorial

$n! = n \cdot (n-1) \cdot (n-2) \cdots 1$. Convention: $0! = 1$.

Growth:

| $n$ | $n!$ |
|---|---|
| 5 | 120 |
| 10 | 3,628,800 |
| 20 | $\approx 2.4 \times 10^{18}$ |
| 70 | $> 10^{100}$ |
| 170 | overflow in float64 |

Stirling's approximation: $n! \approx \sqrt{2\pi n}(n/e)^n$.

### Simple permutation

$P_n = n!$ — ways to order $n$ distinct objects in a line.

### Permutation with repetition

For $n$ objects with $n_1$ of type 1, $n_2$ of type 2, ..., $n_k$ of type $k$:
$$P_n^{n_1, n_2, \ldots, n_k} = \frac{n!}{n_1! \cdot n_2! \cdots n_k!}$$

Anagrams of "ARARA" (3 A's, 2 R's): $5!/(3! \cdot 2!) = 10$.

### Simple arrangement

$$A_n^p = n(n-1)(n-2) \cdots (n-p+1) = \frac{n!}{(n-p)!}$$

Ways to order $p$ objects selected from $n$ available.

### Circular permutation

$n$ objects in a circle: $(n-1)!$. Reason: the "first position" is arbitrary.

### Difference between permutation and arrangement

- **Permutation**: uses all $n$ objects.
- **Arrangement**: selects $p \leq n$ and orders them.

When $p = n$: arrangement equals permutation.

</Porta>

<Porta nivel="5" titulo="For a 5-year-old">

Imagine you have **3 toys** in a row. In how many ways can you arrange them?

You can place A, B, C. Or A, C, B. Or B, A, C. Or B, C, A. Or C, A, B. Or C, B, A. **6 ways**!

That "6" is $3 \times 2 \times 1$ — factorial of 3, written $3!$. For each new toy, multiply.

</Porta>

<Porta nivel="10" titulo="For a 10-year-old">

**Permutation** = shuffle the order of an entire group.

Example: you have 5 books and want to arrange them on the shelf. **How many ways?** $5! = 5 \times 4 \times 3 \times 2 \times 1 = 120$.

**Arrangement** is similar, but you only use **some** of the items, not all.

Example: you have 10 athletes and award a podium (1st, 2nd, 3rd). **$10 \times 9 \times 8 = 720$**.

**Caution**: in both, **order matters**. Swapping A and B on the podium is different.

**When letters repeat**: anagrams of PARA = $4!/2! = 12$ (because A repeats 2 times — divide).

**Round table** is special: 5 people around the table = $4! = 24$ (not $5!$). Why? Rotating everyone together doesn't count as a new arrangement.

</Porta>

<Porta nivel="15" titulo="For a 15-year-old teen">

**Permutation** = order everything. How many ways to organize 5 books on a shelf? $5! = 120$.

**Arrangement** = order a selection. How many ways to award 1st, 2nd, and 3rd place among 10 athletes? $A_{10}^3 = 10 \cdot 9 \cdot 8 = 720$.

**To use the formula directly**: in both, ORDER MATTERS. Swapping A and B in 1st and 2nd is different.

**Permutation with repetition** is special: anagrams of "AMOR" are $4! = 24$, but anagrams of "AMA" are $3!/2! = 3$ (not $6$, because AA equals AA).

**Classic cases**:
- "MISSISSIPPI" (11 letters: 1M, 4I, 4S, 2P): $11!/(1!\cdot 4!\cdot 4!\cdot 2!) = 34\,650$.
- Round table with 8 people: $7! = 5\,040$.

**Factorial grows fast**: $5! = 120$, $10! \approx 3.6$ million, $20!$ is already greater than $10^{18}$. **That's why permutation problems are "explosive"**.

**Arrangement vs FCP**: $A_n^p = n(n-1)(n-2)\cdots(n-p+1)$ is just FCP with the "no repetition" constraint.

</Porta>

<Porta nivel="25" titulo="For an engineering student">

Permutations and arrangements are combinatorial primitives for enumeration and probabilistic estimation.

### Proof: $P_n = n!$

FCP: 1st position $n$ options, 2nd position $n-1$, ..., last 1. Product: $n!$.

### Proof: arrangement via FCP

$A_n^p = n(n-1)(n-2)\cdots(n-p+1) = n!/(n-p)!$.

### Factorial growth

Stirling: $\ln(n!) = n\ln n - n + O(\ln n)$. Hence $n! \sim \sqrt{2\pi n}(n/e)^n$. In ML, important for Gaussian log-likelihood, entropy of categorical models.

### Cross-domain

- **TSP (traveling salesman)**: $n!$ routes → NP-hard. Heuristics (2-opt, Lin-Kernighan, Concorde) tackle practical cases.
- **Card shuffling**: $52! \approx 8\times 10^{67}$ — Riffle shuffle takes $\log_2(52!) \approx 226$ riffles to mix uniformly (Bayer-Diaconis: 7 riffles suffice for "practically" uniform).
- **Sorting**: worst-case comparisons for sort = $\Omega(n \log n)$ because the decision tree has $\geq n!$ leaves.
- **Bioinformatics**: sequence alignment involves permutations; substitution-cipher cryptography is a permutation of the alphabet.

### Permutation as element of $S_n$

$S_n$ is the symmetric group of order $n!$. Group structure — composition, cycles, sign — is central in abstract algebra, Galois theory, physics.

- **Sign**: $\text{sgn}(\sigma) \in \{+1, -1\}$ via parity of the number of transpositions.
- **Cycle decomposition**: unique up to order.
- **Appears in $\det$**: $\det A = \sum_\sigma \text{sgn}(\sigma) \prod a_{i,\sigma(i)}$.

### Applications in ML/quant

- **Permutation tests**: statistical significance without assuming normality.
- **Random forests**: subsampling features and data via permutations.
- **Cross-validation**: shuffle splits.
- **MCMC**: many chains traverse $S_n$ (TSP via simulated annealing).

</Porta>

### Structure of $S_n$

- $|S_n| = n!$.
- **Generators**: adjacent transpositions $(i, i+1)$.
- $S_n$ is non-abelian for $n \geq 3$.
- **Alternating subgroup** $A_n$: permutations of sign +1, $|A_n| = n!/2$.
- $S_n$ is non-solvable for $n \geq 5$ — hence the impossibility of a general formula for the roots of degree ≥ 5 polynomials (Galois).

### Quant — pricing

- **Hidden Markov Models**: hidden states form an alphabet, observations depend on them. Permutation of labels is a model symmetry.
- **Stochastic discount factor**: enumeration of historical scenarios via permutation for bootstrap.

### Permutations in modern ML

- **Transformer attention** is permutation-equivariant in the tokens — hence position encoding.
- **Permutation-invariant networks**: DeepSets, PointNet — aggregate features via sum/max.
- **Sinkhorn-Knopp**: learns a smooth permutation via doubly stochastic matrices.

### Combinatorial generation

- **Lehmer code**: bijection $S_n \leftrightarrow \prod \{0, \ldots, n-1\}$. Allows random indexing without generating the list.
- **Heap's algorithm**: generates all $n!$ permutations in $O(n!)$ total time.
- **Steinhaus-Johnson-Trotter**: permutations in an order that swaps 2 adjacent elements at a time (Gray code for $S_n$).

### Research

- **Pattern avoidance** in permutations — Wilf-equivalence, Stanley-Wilf conjecture.
- **Randomization**: card shuffling, cutoff phenomenon (Diaconis).
- **Permutation polytopes** — geometry.

</Porta>

<Porta nivel="pratica" titulo="Practical application">

**Workflow for permutation/arrangement.**

1. **Order matters?** If yes, this is the place.
2. **Uses all elements?** Permutation. **Subset?** Arrangement.
3. **Has repetition?** Divide by the factorials of multiplicities.
4. **Circular configuration?** $(n-1)!$.

### Python recipe

```python
import math
import itertools

# Factorial
math.factorial(10)        # 3628800

# Arrangement A(n, p)
math.perm(10, 3)          # 720

# Generated permutations
list(itertools.permutations([1, 2, 3]))   # 6 tuples

# Permutations of size k
list(itertools.permutations([1, 2, 3, 4], 2))   # A(4,2) = 12

# Permutation with repetition: generate and remove duplicates
def perm_with_rep(letters):
    return set(itertools.permutations(letters))

print(len(perm_with_rep("ARARA")))    # 10

# Stirling for large factorial
def stirling(n):
    import math
    return math.sqrt(2*math.pi*n) * (n/math.e)**n
```

### SymPy

```python
import sympy as sp
sp.factorial(20)              # exact, 2.4e18
sp.binomial(10, 3) * sp.factorial(3)   # arrangement via combination × p!
```

### Round table and similar

```python
def round_table(n):
    return math.factorial(n - 1)

print(round_table(8))   # 5040
```

### Pitfalls

- **`A == B` in string** vs **`A == B` in multiset**: anagrams with identical letters need special treatment.
- **0 vs 1 indexing** in permutations.
- **Overflow**: $n! > 10^{300}$ for $n > 170$. Use `math.factorial` (Python arbitrary-precision int) or logarithms.
- **"At least one together"** is usually easier via the complement.
